<objective>
Make chargebacks target the oldest OPEN payroll period for the agent (instead of most recent/manual selection) and support per-product chargebacks with partial commission calculation.

Purpose: Currently manual clawbacks grab the first payroll entry (most recent), which is wrong -- chargebacks should hit the oldest unpaid period. Additionally, sales with multiple products need partial chargeback support (chargeback specific products, not all-or-nothing).

Output: Updated schema with ClawbackProduct join table, backend logic for oldest-period targeting and per-product commission calculation, UI with product selection in manual chargeback form and auto-period in alert approval.
</objective>

    **2. Helper: findOldestOpenPeriodForAgent**

    Add to `apps/ops-api/src/services/payroll.ts` a new exported function:

    ```typescript
    export async function findOldestOpenPeriodForAgent(agentId: string): Promise<string | null> {
      const period = await prisma.payrollPeriod.findFirst({
        where: {
          status: "OPEN",
          entries: { some: { agentId } },
        },
        orderBy: { weekStart: "asc" },
        select: { id: true },
      });
      return period?.id ?? null;
    }
    ```

    **3. Helper: calculatePerProductCommission**

    Add to `apps/ops-api/src/services/payroll.ts` a function that, given a sale (with product + addons) and an array of productIds to chargeback, returns the commission amount for JUST those products. Logic:
    - Load the sale with product, addons (each with product relation)
    - If productIds is empty or includes ALL products (core + all addons), return the full payroll entry payout amount (existing behavior)
    - Otherwise, calculate the commission contribution of only the selected products:
      - For CORE product: use the core rate on the core premium (from calculateCommission logic -- threshold/above/below)
      - For ADDON with bundledCommission=null: use the core rate on the addon premium (bundled into core)
      - For ADDON with bundledCommission set: use bundledCommission rate on addon premium
      - For AD_D: use bundledCommission rate on addon premium
      - For ACA_PL: use flatCommission * memberCount
    - Apply enrollment fee rules only if CORE product is included in chargeback
    - Apply halving only if original sale had halvingReason and was not approved
    - Round to 2 decimals

    **4. Update POST /clawbacks route (payroll.ts:171-203)**

    After finding the sale, change the period targeting:

    - If no OPEN period found, find the agent's most recent payroll entry for this sale and use its periodId (fallback for edge case where all periods are locked)
    - Include the sale's product + addons in the query (add `product: true, addons: { include: { product: true } }` to the Sale include)

    Calculate chargeback amount:
    - If `productIds` provided and not empty: call `calculatePerProductCommission(sale, productIds)` for partial amount
    - If `productIds` not provided or empty: use full payroll entry netAmount (existing behavior)

    Create clawback with `appliedPayrollPeriodId` set to the found period.

    - Notes should list which products: `"Partial chargeback: [product names]"`

    For the PayrollEntry update (adjustmentAmount deduction), apply the chargeback amount to the entry in the target period (not necessarily the first entry). Find the PayrollEntry matching `{ payrollPeriodId: targetPeriodId, saleId: sale.id }`.
