<objective>
GAP-46-UAT-02: Plan 46-02 had the design inverted. Alerts were originally a CS → payroll review queue, NOT a universal pre-clawback step. Plan 46-02 wired createAlertFromChargeback into the shared POST /api/chargebacks handler and fired it for ALL submissions. The user has clarified: payroll-side chargeback submissions should go DIRECTLY to clawback because payroll IS the team that would otherwise be approving the alert — the alert step is redundant and noisy. CS-side submissions are the ONLY path that needs to create alerts (so payroll can review what CS submitted before it sticks).

Purpose: Add a `source: "CS" | "PAYROLL"` discriminator to POST /api/chargebacks. Gate the createAlertFromChargeback call on `source === "CS"`. Update both UI submitters to declare their source explicitly. Default the server to "PAYROLL" if source is missing (safer — no alert created for legacy/unauthored callers).
Output: One field on the Zod schema, one if-gate on the alert loop, two UI body changes.

Note: This plan depends on 46-06 — the alert wire-up must actually fire reliably before this plan can gate it. Run 46-06 first.
</objective>
