## Interaction Contracts

### 1. KPI Counter Bar (TRKC-01)

**Layout:** 4-column grid (`grid-template-columns: repeat(4, 1fr)`, gap: `spacing[4]`).

| Counter | Label | Color | Source |
|---------|-------|-------|--------|
| 1 | TOTAL CHARGEBACKS | `colors.danger` | `/api/chargebacks/totals` -- sum of absolute chargeback_amount |
| 2 | TOTAL RECOVERED | `colors.success` | Hardcoded $0.00 until Phase 15 |
| 3 | NET EXPOSURE | `colors.danger` when positive, `colors.success` when zero | Total Chargebacks minus Total Recovered |
| 4 | RECORDS | `colors.textPrimary` | `/api/chargebacks/totals` -- total record count |

Behavior:
- All counters use `AnimatedNumber` with `duration={600}`, `decimals={2}`, `prefix="$"` (except Records which uses no prefix and `decimals={0}`)
- Counters show global totals -- never affected by search or filter state
- Counter 3 subtitle: "chargebacks - recovered"
- Animation triggers on initial load and on data refetch (after delete)

### 2. Pending Terms Summary Bar (TRKT-01)

**Layout:** Horizontal strip below KPI bar, inside a single `Card`. Uses a flex row with items spaced evenly.

| Metric | Label | Color | Computation |
|--------|-------|-------|-------------|
| Total Pending | "TOTAL PENDING" | `colors.textPrimary` | Count of all pending terms records |
| By Hold Reason | "{reason}" | `colors.textSecondary` | Count per unique `holdReason` value |
| Urgent | "DUE WITHIN 7 DAYS" | `colors.danger` | Count where `nextBilling` is within 7 days from today |

Behavior:
- Summary computed from full unfiltered pending terms dataset
- Hold reason categories rendered as small pill/badge items showing "{reason}: {count}"
- Urgent count rendered in danger color with bold weight
- If no pending terms exist, show `EmptyState` instead

### 3. Search Bar (TRKC-04, TRKT-05)

**Layout:** Full-width input with `Search` icon prefix, positioned between summary bar and tables.

| Property | Value |
|----------|-------|
| Placeholder | "Search by name, ID, company, phone..." |
| Style | `baseInputStyle` with `Search` icon at left (16px padding-left for icon space) |
| Debounce | None (filter on every keystroke -- dataset is 200 records max) |
| Clear | `X` icon button appears when search term is non-empty |

Behavior:
- Single search box filters BOTH chargeback and pending terms tables simultaneously
- Chargeback search fields: payeeName, memberAgentCompany, memberId, memberAgentId
- Pending terms search fields: memberName, memberId, agentName, agentId, phone
- Case-insensitive partial match
- Tables show filtered results; KPI bar and summary bar remain unaffected

### 4. Filter Panel (TRKC-03, TRKT-04)

**Toggle:** `Button` with `variant="ghost"`, icon `Filter`, label "Filters", chevron indicator. Positioned inline-right of search bar.

**Panel:** Renders below search bar when expanded. Background: `colors.bgSurfaceInset`, border-radius: `radius.lg`, padding: `spacing[4]`.

**Chargeback filters** (grid layout, `repeat(auto-fill, minmax(200px, 1fr))`):

| Filter | Input type | Field |
|--------|-----------|-------|
| Date range (from) | date input | `postedDate >= value` |
| Date range (to) | date input | `postedDate <= value` |
| Product | text input | `product` partial match |
| Member Company | text input | `memberCompany` partial match |
| Member Agent Company | text input | `memberAgentCompany` partial match |
| Amount min | number input | `chargebackAmount >= value` |
| Amount max | number input | `chargebackAmount <= value` |

**Pending terms filters** (same grid layout):

| Filter | Input type | Field |
|--------|-----------|-------|
| Agent | text input | `agentName` partial match (behind-the-scenes field) |
| State | text input | `state` partial match |
| Product | text input | `product` partial match |
| Hold Reason | text input | `holdReason` keyword match |
| Date range (from) | date input | `holdDate >= value` or `nextBilling >= value` |
| Date range (to) | date input | `holdDate <= value` or `nextBilling <= value` |

Behavior:
- All filtering is client-side (200-record cap per endpoint)
- Filter labels use `baseLabelStyle` (11px, uppercase, caps tracking)
- Filter inputs use `baseInputStyle`
- Filters apply to their respective table only (chargeback filters do not affect pending terms and vice versa)
- A "Clear Filters" text button appears when any filter is active

### 5. Chargeback Table (TRKC-02, TRKC-05)

**Columns** (8 data columns + 1 action column):

| # | Header | Field | Width | Style override |
|---|--------|-------|-------|----------------|
| 1 | Date Posted | `postedDate` | auto | Format: `M/D/YYYY` via split pattern |
| 2 | Member | `memberCompany` | auto | Truncate with ellipsis at 200px max-width |
| 3 | Member ID | `memberId` | 110px | -- |
| 4 | Product | `product` | auto | Truncate with ellipsis at 200px max-width |
| 5 | Type | `type` | auto | -- |
| 6 | Total | `chargebackAmount` | auto | Always `colors.danger`, format: `$X,XXX.XX` |
| 7 | Assigned To | `assignedTo` | auto | Show "Unassigned" when null |
| 8 | Submitted | `submittedAt` | auto | Format: `M/D/YYYY` |
| 9 | (delete) | -- | 50px | `X` icon, `colors.textMuted` default, `colors.danger` on hover |

Sorting:
- All 8 data columns are sortable (click header to toggle ascending/descending)
- Sort indicator: `ChevronUp` (12px) for ascending, `ChevronDown` (12px) for descending, shown inline after label text with 4px gap
- Only active sort column shows indicator
- Default sort: `submittedAt` descending (newest first)
- Header cursor: `pointer`, `userSelect: "none"`

Empty state:
- `EmptyState` with `title="No Chargebacks Yet"` and `description="Chargeback records will appear here once submissions are processed."`

Filtered empty state:
- `EmptyState` with `title="No Results"` and `description="No chargebacks match your current search or filters. Try adjusting your criteria."`

### 6. Pending Terms Table (TRKT-02, TRKT-03, TRKT-06)

**Columns** (7 data columns + 1 action column):

| # | Header | Field | Style override |
|---|--------|-------|----------------|
| 1 | Member Name | `memberName` | -- |
| 2 | Member ID | `memberId` | -- |
| 3 | Phone | `phone` | -- |
| 4 | Product | `product` | Truncate with ellipsis at 200px max-width |
| 5 | Hold Date | `holdDate` | `color: colors.danger`, format: `M/D/YYYY` via split pattern |
| 6 | Next Billing | `nextBilling` | `color: colors.success`, format: `M/D/YYYY` |
| 7 | Assigned To | `assignedTo` | Show "Unassigned" when null |
| 8 | (delete) | -- | `X` icon, same hover pattern as chargeback delete |

Hidden fields (TRKT-03): `agentName` and `agentId` are used for grouping and search but never rendered as table columns.

**Group-by-agent (TRKT-06):**
- Records are always grouped by `agentName` (no flat-list toggle)
- Group header: full-width row spanning all columns, styled as:
  - Background: `colors.bgSurfaceInset`
  - Font: 13px, weight 600 (semibold), color `colors.textPrimary`
  - Padding: `12px 14px` (matching `baseTdStyle`)
  - Cursor: `pointer`
  - Left icon: `ChevronRight` (14px) when collapsed, `ChevronDown` (14px) when expanded
  - Label: `"{agentName} ({count})"` where count is records in that group
  - Default to `"Unassigned"` when `agentName` is null
- All groups start expanded by default
- Clicking group header toggles collapse/expand for that group only
- Collapse state tracked via `Set<string>` of collapsed agent names

Sorting:
- All 7 data columns are sortable (same interaction as chargeback table)
- Sort applies within each agent group (records inside a group are sorted, group order is alphabetical by agent name)
- Default sort: `holdDate` descending (most recent holds first)

Empty state:
- `EmptyState` with `title="No Pending Terms Yet"` and `description="Pending terms records will appear here once submissions are processed."`

Filtered empty state:
- `EmptyState` with `title="No Results"` and `description="No pending terms match your current search or filters. Try adjusting your criteria."`

### 7. CSV Export Button (TRKC-06, TRKT-07)

**Visibility:** Only rendered when user role includes `SUPER_ADMIN` or `OWNER_VIEW`. Role fetched from `GET /api/session/me` on mount.

**Placement:** Top-right of the Tracking tab, inline with the search bar row.

**Style:**
- `Button` with `variant="ghost"`
- Icon: `Download` (14px) with label "Export CSV"
- Gap: 6px between icon and text

**Behavior:**
- Exports filtered/searched data (what you see is what you export)
- Single CSV file with both chargeback and pending terms sections
- Filename: `cs-tracking-YYYY-MM-DD.csv`
- CSV structure:
  ```
  --- CHARGEBACKS ---
  Date Posted,Member,Member ID,Product,Type,Total,Assigned To,Submitted
  {filtered chargeback rows}

  --- PENDING TERMS ---
  Member Name,Member ID,Phone,Product,Hold Date,Next Billing,Assigned To
  {filtered pending terms rows}
  ```
- Values containing commas or quotes are escaped using the `esc()` helper pattern from payroll dashboard
- Uses `Blob` + `URL.createObjectURL` + anchor click pattern (existing project convention)
