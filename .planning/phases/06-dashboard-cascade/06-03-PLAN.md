**3. Create the onSaleChanged callback:**
The payroll dashboard shows payroll entries grouped by agent within the currently selected period. On `sale:changed`:

- Check if any `payrollEntries` in the payload match the currently selected period (compare `periodId` with the selected period's ID)
- If the period matches, refetch the payroll data for the current period to get accurate totals
- Highlight the new entry IDs from the payload

The payroll dashboard has complex state with agent cards, entries, headers, bonuses, etc. Per the same rationale as RESEARCH.md Open Question 2 (aggregation complexity), the safest approach is to refetch the current period's data rather than incrementally patch.

```typescript
const handleSaleChanged = useCallback((payload: SaleChangedPayload) => {
  // Check if any payroll entry belongs to the currently selected period
  const matchesPeriod = payload.payrollEntries.some(
    (e) => e.periodId === selectedPeriod?.id
  );
  if (matchesPeriod) {
    // Highlight the new entries
    payload.payrollEntries.forEach((e) => highlightEntry(e.id));
    // Refetch period data to get accurate agent card totals
    fetchPayrollData(); // or whatever the existing fetch function is called
  }
}, []);
```

Note: The `selectedPeriod?.id` reference inside a useCallback needs careful handling to avoid stale closures. Since useSocket uses refs internally for the callback, the latest `handleSaleChanged` is always called. Alternatively, use a ref for selectedPeriod.
