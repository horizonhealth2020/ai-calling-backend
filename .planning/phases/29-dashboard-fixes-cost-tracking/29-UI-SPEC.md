# Phase 29 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom token system) |
| Preset | not applicable |
| Component library | none (inline React.CSSProperties) |
| Icon library | lucide-react |
| Font | Inter (--font-sans), JetBrains Mono (--font-mono) |

Source: `packages/ui/src/tokens.ts` and `packages/ui/src/theme.css` define all design tokens. All components use inline `React.CSSProperties` objects referencing these tokens. No shadcn, no Tailwind, no CSS modules.

---

## Spacing Scale

Declared values from `packages/ui/src/tokens.ts` `spacing` object:

| Token | Value | Usage |
|-------|-------|-------|
| 1 | 4px | Icon gaps, inline padding |
| 2 | 8px | Compact element spacing, button gaps |
| 3 | 12px | Minor section padding |
| 4 | 16px | Default element spacing |
| 5 | 20px | Medium element spacing |
| 6 | 24px | Card padding (baseCardStyle), section padding |
| 8 | 32px | Layout gaps |
| 10 | 40px | Large layout spacing |
| 12 | 48px | Major section breaks |
| 16 | 64px | Page-level spacing |

Exceptions: The spacing scale includes non-power-of-2 multiples of 4 (12px, 20px, 40px). This is the established project convention and must be followed. Table cells use `padding: "12px 14px"` (baseTdStyle) and `padding: "10px 14px"` (baseThStyle) -- these are the standard table spacing values.

### DATA-03/04/05: Cost Display Logic (ManagerTracker.tsx, OwnerOverview.tsx)

| State | Lead Spend Display | Cost Per Sale Display |
|-------|-------------------|----------------------|
| Convoso not configured (no auth token) | "\u2014" in `--text-muted` | "\u2014" in `--text-muted` |
| Convoso configured, agent has zero calls | "$0.00" in `--text-secondary` | "\u2014" in `--text-muted` |
| Agent has calls, zero sales | `formatDollar(totalLeadCost)` in `--text-primary` | "\u2014" in `--text-muted` |
| Agent has calls and sales | `formatDollar(totalLeadCost)` in `--text-primary` | `formatDollar(costPerSale)` in `--text-primary` |

The em-dash character is U+2014 ("\u2014"), not a hyphen. Font: 13px, displayed in table cells using baseTdStyle.

### CS-02/03/04: CS Resolved Log Table (CSResolvedLog.tsx)

**Filter Bar:**

| Element | Spec |
|---------|------|

| Layout | Filters wrap on narrow screens (flex-wrap: wrap) |

**Table:**

| Column | Width | Content | Alignment |
|--------|-------|---------|-----------|
| Type | 120px | Badge: "Chargeback" or "Pending Term" | Left |
| Agent | auto | Agent name string | Left |
| Member | auto | Member name string | Left |
| Resolution Date | 140px | Formatted date (e.g., "Mar 25, 2026") | Left |
| Resolved By | 140px | Resolver user name | Left |
| Resolution Note | auto (flex: 1) | TruncatedNote component (see below) | Left |
| Original Amount | 120px | `formatDollar(amount)` | Right |

**Type Badge:**

| Type | Text Color | Background |
|------|-----------|------------|
| Chargeback | `--danger` (#f87171) | `--danger-bg` (rgba(248,113,113,0.08)) |
| Pending Term | `--warning` (#fbbf24) | `--warning-bg` (rgba(251,191,36,0.08)) |

Badge style: 11px, weight 600, padding 4px 10px, border-radius 9999px (pill shape).

**TruncatedNote Component:**

| Element | Spec |
|---------|------|
| Truncation threshold | 80 characters |
| Collapsed display | First 80 chars + "..." -- 13px, color `--text-secondary` |
| Expanded display | Full text -- 13px, color `--text-secondary` |
| Interaction | Click to toggle expand/collapse |
| Cursor | `pointer` when text exceeds threshold |
| Title attribute | "Click to expand" / "Click to collapse" |
| Null note | "\u2014" in `--text-muted` |

**Sort:**

| Default sort | Resolution Date descending (most recent first) |
|--------------|-----------------------------------------------|
| Sort interaction | None -- static sort order per D-15 |

**Loading State:**

| Element | Spec |
|---------|------|

**Empty State:**

| Element | Spec |
|---------|------|

**Error State:**

| Element | Spec |
|---------|------|

## Interaction States

### Existing Patterns (reuse as-is)

| Pattern | Source | Used By |
|---------|--------|---------|
| Table row hover | `--row-hover-bg: rgba(255,255,255,0.02)` via CSS | Products table, Resolved Log table |
| Input focus | `border-color: --border-focus` (teal) + `box-shadow: --shadow-glow-primary` | Buffer field, agent filter input |
| Tab active | 2px bottom border in `--accent-teal` | Resolved Log tab |
| Button hover | `opacity: 0.85` on hover | No primary buttons in this phase (read-only views) |

### New Interactions

| Interaction | Spec |
|-------------|------|
| TruncatedNote click | Toggle between truncated (80 chars + "...") and full text. No animation. Cursor changes to pointer. |
| Type filter change | Re-fetches data from API with `?type=` query param. Table shows loading state during fetch. |
| Date range change | Re-fetches data from API with `?from=&to=` query params. Uses DateRangeFilter built-in behavior. |
| Agent filter input | Debounce 300ms, then re-fetch with `?agentName=` query param. |

## Accessibility Notes

| Requirement | Implementation |
|-------------|----------------|
| Table headers | Use `<th>` with `scope="col"` for all table header cells |
| Type badges | Include aria-label matching visual text |
| TruncatedNote | Use `aria-expanded` attribute reflecting current state |
| Filter labels | Each filter input has an associated `<label>` (can be visually hidden for inline layouts) |
| Em-dash placeholders | Use `aria-label="No data"` on em-dash spans for screen readers |
| Number input | Buffer field uses `type="number"` with `min="0"` for native validation |
