## Typography

All values from `packages/ui/src/tokens.ts` typography object. Phase 11 uses 4 sizes:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (base) | 400 (normal) | 1.6 |
| Label | 11px (xs) | 700 (bold) | 1.45 |
| Heading | 18px (lg) | 700 (bold) | 1.4 |
| Subtitle | 13px (sm) | 500 (medium) | 1.5 |

Font family: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`

**Phase 11 specific usage:**
- Page title ("Customer Service"): 18px bold, `colors.textPrimary` -- matches `PAGE_TITLE` in PageShell
- Page subtitle ("Chargebacks & Pending Terms"): 11px medium, `colors.textTertiary` -- matches `PAGE_SUBTITLE`
- Sidebar nav labels: 13px medium/semibold -- matches existing `SidebarNavItem` labelStyle
- Placeholder content headings inside tabs: 14px normal, `colors.textSecondary`

**Source:** packages/ui/src/tokens.ts `typography` object, packages/ui/src/index.tsx PageShell styles

## Color

All colors are CSS custom properties defined in `packages/ui/src/theme.css`, consumed via `colors` token object. The system supports dark and light themes automatically.

| Role | Value (dark mode) | Usage |
|------|-------------------|-------|
| Dominant (60%) | `--bg-root` (#070a0a) | Page background |
| Secondary (30%) | `--bg-surface` (#0c1414) | Sidebar, topbar, cards |
| Accent (10%) | `--primary-500` (#14b8a6, teal) | Active nav indicator, logo badge glow, focus ring |
| Destructive | `--danger` (#f87171) | Not used in Phase 11 (no destructive actions) |

Accent reserved for:
- Sidebar active nav indicator (3px left border, `colors.primary500`)
- Active nav icon color (`colors.primary400`)
- Logo badge gradient background (`colors.accentGradient`)
- Focus-visible outline ring (`--border-focus`)
- DASHBOARD_MAP card for CUSTOMER_SERVICE: amber gradient (#f59e0b to #d97706) for visual distinction from other role cards

Semantic colors available but not actively used in Phase 11 placeholder content:
- `--success` (#34d399) -- reserved for Phase 14 (recovered amount)
- `--danger` (#f87171) -- reserved for Phase 14 (chargeback amounts in red)
- `--warning` (#fbbf24) -- reserved for Phase 15 (urgent pending terms)

**Source:** packages/ui/src/theme.css, packages/ui/src/tokens.ts `colors` object

## Component Inventory

Phase 11 uses only existing @ops/ui components. No new components are created.

| Component | Source | Usage in Phase 11 |
|-----------|--------|-------------------|
| `PageShell` | `@ops/ui` | Dashboard shell with sidebar nav, topbar, content area |
| `Card` | `@ops/ui` | Two Card sections per tab for section grouping |
| `EmptyState` | `@ops/ui` | Placeholder content inside each Card section |

### Nav Items Configuration

```
navItems = [
  { icon: <ClipboardList size={18} />, label: "Submissions", key: "submissions" },
  { icon: <BarChart3 size={18} />, label: "Tracking", key: "tracking" },
]
```

Icons from lucide-react: `ClipboardList` for Submissions, `BarChart3` for Tracking.

### Tab Content Layout

**Submissions tab:** Two `Card` components stacked vertically with `spacing[6]` (24px) gap between them. Each card contains an `EmptyState` with the placeholder copy defined above.

**Tracking tab:** Two `Card` components stacked vertically with `spacing[6]` (24px) gap between them. Each card contains an `EmptyState` with the placeholder copy defined above.

**Source:** 11-CONTEXT.md ("Submissions tab shows two separate Card sections"), 11-RESEARCH.md Pattern 1

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Tab switching | Click sidebar nav item to toggle between "submissions" and "tracking" via `useState<Tab>` |
| Default tab | "submissions" on initial load |
| Auth token capture | `captureTokenFromUrl()` runs once in `useEffect` on mount |
| Mobile nav | Bottom nav bar appears at viewport < 1024px (handled by PageShell) |
| Hover states | Sidebar nav items show raised background on hover (handled by PageShell) |
| Active state | Sidebar nav item shows 3px teal left border + primary icon color (handled by PageShell) |
| Keyboard nav | Enter/Space on nav items triggers tab switch (handled by PageShell) |

No data fetching, no API calls, no loading states in Phase 11. All content is static placeholder.

**Source:** 11-RESEARCH.md Pattern 1, packages/ui/src/index.tsx SidebarNavItem
