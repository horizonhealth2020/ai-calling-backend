<objective>
Fix Bug 1: ACA sales entry into commission. Currently when a manager submits a parent sale with the ACA checkbox toggled, the user perceives that only the ACA covering row appears in payroll (while the parent sale's row is actually present but visually disconnected), and the ACA row renders verbose `$X.XX x N members = $Y` text instead of a flat dollar commission.

Research confirms this is a display-layer bug: both PayrollEntry rows ARE persisted correctly. The fix is purely view-side — surface `acaCoveringSaleId` in the API response, fold ACA covering entries into their parent entries on the client, render a single unified row with an ACA badge, and remove the verbose commission string from both dashboard and print.

Purpose: Agents and payroll staff need to see exactly one row per member sale with the correct total commission, matching the pattern every other product uses.

Output: Unified ACA rendering on dashboard and print. No schema/data migration.
</objective>
