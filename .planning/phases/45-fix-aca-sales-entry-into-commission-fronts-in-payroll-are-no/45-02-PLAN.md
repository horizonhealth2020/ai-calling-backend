<objective>
Fix Bug 2: Fronts entered on the agent-period adjustment level are not reliably carrying into the next period as `Fronted Hold` across the unlock → edit → re-lock workflow. Research confirmed Path A (idempotency on re-lock) is definitively broken: `carryoverExecuted` is set at `carryover.ts:84` but never reset on unlock, so re-locking silently skips `executeCarryover`. Path B (first lock) is intact and covered by existing tests.

This plan adds a deterministic reversal mechanism: stores the exact carried amount on the next-period adjustment row via a new `carryoverAmount` field, exports a new `reverseCarryover(periodId)` helper, and wires it into the PATCH status handler so LOCKED→OPEN transitions reverse the prior carryover before flipping the period state.

Purpose: Payroll needs to unlock, edit fronts, and re-lock without silent data loss.

Output: Migration + carryover service update + route wiring + two new test cases.
</objective>

    **2a. carryover.ts — add `reverseCarryover` export.**

    Add a new exported async function below `executeCarryover`:
    ```ts
    export async function reverseCarryover(sourcePeriodId: string): Promise<{ reversed: number; rowsTouched: number }> {
      return prisma.$transaction(async (tx) => {
        // Load the source period to verify state
        const source = await tx.payrollPeriod.findUnique({ where: { id: sourcePeriodId } });
        if (!source || !source.carryoverExecuted) {
          return { reversed: 0, rowsTouched: 0 };
        }

        // Find all next-period adjustments that originated from this source period
        const targets = await tx.agentPeriodAdjustment.findMany({
          where: {
            carryoverSourcePeriodId: sourcePeriodId,
            holdFromCarryover: true,
          },
        });

        let totalReversed = 0;
        for (const row of targets) {
          const carriedAmount = Number(row.carryoverAmount ?? 0);
          if (carriedAmount <= 0) continue;

          const currentHold = Number(row.holdAmount);
          const newHold = Math.max(0, currentHold - carriedAmount);
          totalReversed += carriedAmount;

          if (newHold === 0) {
            // Row's hold came solely from carryover — clear all carryover metadata.
            await tx.agentPeriodAdjustment.update({
              where: { id: row.id },
              data: {
                holdAmount: 0,
                holdFromCarryover: false,
                holdLabel: null,
                carryoverSourcePeriodId: null,
                carryoverAmount: null,
              },
            });
          } else {
            // Partial reversal — hold had other contributions; keep metadata but zero carryoverAmount.
            await tx.agentPeriodAdjustment.update({
              where: { id: row.id },
              data: {
                holdAmount: newHold,
                carryoverAmount: null,
              },
            });
          }
        }

        // Reset the source period so re-lock can carryover again from a clean slate
        await tx.payrollPeriod.update({
          where: { id: sourcePeriodId },
          data: { carryoverExecuted: false },
        });

        return { reversed: totalReversed, rowsTouched: targets.length };
      });
    }
    ```
