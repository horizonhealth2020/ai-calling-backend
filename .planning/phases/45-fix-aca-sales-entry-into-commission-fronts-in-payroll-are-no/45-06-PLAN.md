<objective>
Close GAP-45-06. UAT Test 4 failed: the manager/agent sales tracking view (`apps/ops-dashboard/app/(dashboard)/manager/ManagerSales.tsx`) renders ACA-attached sales as TWO rows — the parent (e.g., "ACA test (686905239) / Complete Care Max / $139.98") AND the ACA covering child ("ACA test / Blue Cross Blue Shield / $0.00") — and the per-agent "N sales" badge counts both rows.

**Diagnosis (from reading ManagerSales.tsx L347-L352 and routes/sales.ts L309-L334):**

Plan 45-01 only added the ACA fold to the payroll dashboard surface (`PayrollPeriods.tsx` / `WeekSection.tsx` / `PayrollExports.tsx`). The manager sales tracking view is a completely different component that fetches `${API}/api/sales?range=week` (sales.ts:309 GET handler) and groups raw rows by agent name with no fold logic at all (`byAgent` Map, L347-L352).

The API endpoint already returns `acaCoveringSaleId` automatically because `prisma.sale.findMany` at L314 uses `include: { agent, product, leadSource, addons, _count }` — Prisma's `include` syntax preserves all scalar fields on the model, so `acaCoveringSaleId` is already in the response payload. The only missing piece is the local `Sale` type at L38 of ManagerSales.tsx, which does not declare `acaCoveringSaleId`, and the `byAgent` builder which has no fold pass.

Other surfaces consuming `/api/sales` were checked: `apps/ops-dashboard/app/(dashboard)/manager/ManagerTracker.tsx` does NOT render a per-agent sales list (grep for `byAgent|sales.length` returned no matches there), and the only other `/api/sales?range=week` consumer in this component is the same `salesList` state. Out-of-scope surfaces (exports, reports, top-earner badge math) are not present in this view.

**Fix:** Add `acaCoveringSaleId?: string | null` to the local `Sale` type, then apply the same order-independent two-pass fold from Plan 45-04 (Task 1) to the `salesList` (or to the `filtered` array right before `byAgent` is built). The folded sale gets the ACA child's premium added to a synthetic `acaPremium` marker so the per-row Premium column can optionally show the contribution; the parent's existing premium remains unchanged. The "N sales" badge (`sales.length`) automatically reflects the post-fold count because it iterates the folded list.

Purpose: Eliminate visual duplication so managers see one row per logical sale and accurate per-agent counts.

Output: One file modified. No API change (data already present), no schema change, no migration.
</objective>

    ```ts
    const filtered = salesDay === "all"
      ? salesList
      : salesList.filter(s => getDayOfWeek(s.saleDate.slice(0, 10)) === DAYS.indexOf(salesDay as typeof DAYS[number]));

    // ── GAP-45-06: Fold ACA covering child sales into their parent rows ──
    // Mirrors the two-pass fold in PayrollPeriods.tsx (GAP-45-04). The API
    // returns two Sale rows for an ACA-attached entry: the parent (Complete
    // Care, etc.) and the ACA child whose acaCoveringSaleId points back at
    // the parent. Visually we want ONE row per logical sale and the per-agent
    // 'N sales' badge to count logical (post-fold) sales.
    //
    // Pass 1: index ACA child sales by parent id.
    const acaChildrenByParentId = new Map<string, Sale[]>();
    for (const s of filtered) {
      if (s.acaCoveringSaleId) {
        if (!acaChildrenByParentId.has(s.acaCoveringSaleId)) acaChildrenByParentId.set(s.acaCoveringSaleId, []);
        acaChildrenByParentId.get(s.acaCoveringSaleId)!.push(s);
      }
    }
    // Pass 2: emit non-child rows; skip child rows that have a present parent.
    const foldedSales: Sale[] = [];
    for (const s of filtered) {
      if (s.acaCoveringSaleId) continue;
      foldedSales.push(s);
    }
    // Pass 3 (defensive): if an ACA child has no matching parent in the
    // current visible window (orphaned data), surface it as a standalone row
    // so payroll is not silently dropped.
    for (const [parentId, children] of acaChildrenByParentId) {
      const hasParent = filtered.some(s => s.id === parentId && !s.acaCoveringSaleId);
      if (!hasParent) {
        for (const orphan of children) foldedSales.push(orphan);
      }
    }

    const byAgent = new Map<string, Sale[]>();
    for (const s of foldedSales) {
      const name = s.agent.name;
      if (!byAgent.has(name)) byAgent.set(name, []);
      byAgent.get(name)!.push(s);
    }
    ```
