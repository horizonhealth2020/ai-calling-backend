# Phase 18 -- UI Design Contract

> Visual and interaction contract for platform polish & integration. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom token system) |
| Preset | not applicable |
| Component library | @ops/ui (custom: PageShell, Button, Card, StatCard, Badge, TabNav, Toast, EmptyState, Input, Select, Tooltip, Skeleton, AnimatedNumber, ProgressRing) |
| Icon library | Inline SVG (no external icon library) |
| Font | Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif |

**Styling approach:** Inline `React.CSSProperties` constant objects. No Tailwind, no CSS modules. All tokens imported from `@ops/ui` tokens. Dark glassmorphism theme with teal accent gradient. Light/dark theme toggle via CSS custom properties in `theme.css`.

## Component Inventory -- New for Phase 18

### 1. DateRangeFilter (shared @ops/ui)

**Purpose:** Replaces current pill-button date filters across all 6 dashboards.

| Element | Specification |
|---------|---------------|
| Layout | Horizontal row: preset pills + "Custom" toggle + from/to date inputs |
| Preset pills | "Last 7 days", "Last 30 days", "This month" -- use `Button variant="ghost"` for inactive, `variant="primary"` for active |
| Custom date inputs | Two `Input` fields (type="date") with "From" and "To" labels, visible only when "Custom" preset selected |
| Spacing | `gap: spacing[2]` (8px) between pills; `gap: spacing[3]` (12px) between preset group and custom inputs |
| Height | Match existing button height: `padding: 6px 12px` (Button size="sm") |
| Placement | Above export button, right-aligned within card header area |
| State | Controlled: `{ preset: string; from?: string; to?: string }` |

### 2. PayrollAlertTable (payroll-dashboard)

**Purpose:** Chargeback alerts displayed above the current open week.

| Element | Specification |
|---------|---------------|
| Container | Card with `dangerBg` (`rgba(248,113,113,0.08)`) left border accent (4px solid `danger`) |
| Header | "Chargeback Alerts" label (xs style), badge pill with count |
| Table columns | Agent Name, Customer, Amount, Date Submitted, Actions |
| Approve action | `Button variant="success" size="sm"` labeled "Approve" -- opens dropdown of agent's unpaid periods |
| Clear action | `Button variant="ghost" size="sm"` labeled "Clear" -- with confirmation (see Copywriting) |
| Period dropdown | `Select` component listing unpaid period labels (e.g., "Week of Mar 10 - Mar 16") |
| Empty state | "No pending alerts" with checkmark icon |
| Socket.IO | Highlight new rows with `HIGHLIGHT_GLOW` pattern on `alert:created` event |

### 3. AgentKPITable (owner-dashboard)

**Purpose:** Per-agent chargeback and pending term metrics (30-day rolling window).

| Element | Specification |
|---------|---------------|
| Layout | Card with standard `baseCardStyle` |
| Header row | Three `StatCard` components: Total Chargebacks (count), Total Chargeback $ (dollar), Total Pending Terms (count) |
| Table columns | Agent Name, Chargebacks (count), Chargeback Total ($), Pending Terms (count) |
| Sorting | Click column header to sort ascending/descending, default by chargeback total descending |
| Cell formatting | Dollar amounts via `formatDollar()`, counts as plain integers |
| Empty state | "No agent KPI data for the last 30 days" |

### 4. PermissionTable (owner-dashboard)

**Purpose:** Role-level defaults with per-user overrides for create actions.

| Element | Specification |
|---------|---------------|
| Layout | Card containing a matrix table |
| Row headers | One row per user (name + role badge) |
| Column headers | Each configurable create action: Sales, Chargebacks, Pending Terms, Reps, Agents, Products, Lead Sources |
| Cell | Checkbox (custom styled, teal accent when checked) |
| Hard-coded rows | "Payroll Access" and "User Creation" rows shown as locked with lock icon, always SUPER_ADMIN only, non-interactive |
| Override indicator | If user permission differs from role default, show a small dot indicator next to checkbox |
| Save | `Button variant="primary"` labeled "Save Permissions" at bottom-right of card |

### 5. ParsePreviewCard (manager-dashboard)

**Purpose:** Confirmation card showing parsed receipt fields before populating the sale form.

| Element | Specification |
|---------|---------------|
| Layout | Card with `bgSurfaceRaised` background, `borderStrong` border |
| Header | "Parsed Sale Details" with edit icon |
| Content | Key-value pairs: Customer, Products (as Badge pills), State, Status, Lead Source |
| Matched products | Each shown as a `Badge` with product name; unmatched products shown with warning color |
| Actions | "Confirm & Fill" (`Button variant="primary"`), "Discard" (`Button variant="ghost"`) |
| Spacing | `gap: spacing[2]` (8px) between rows, `padding: spacing[4]` (16px) |

### 6. RepChecklist (cs-dashboard)

**Purpose:** Per-rep tracking of assigned chargebacks and pending terms with completion status.

| Element | Specification |
|---------|---------------|
| Layout | Expandable card per rep (click to expand/collapse) |
| Header | Rep name, assigned count, completed count, `ProgressRing` showing completion percentage |
| Expanded content | List of assigned items with checkbox for completion, item type badge (Chargeback/Pending Term) |
| Assignment dropdown | Editable `Select` showing current auto-assigned rep, allows override before submission |

### 7. StorageAlert (owner-dashboard)

**Purpose:** Banner when PostgreSQL storage nears capacity.

| Element | Specification |
|---------|---------------|
| Layout | Full-width banner above main content, `warningBg` background, `warning` left border (4px) |
| Content | Warning icon + "Database storage at {X}% capacity" + "Download CSVs" link + "Manage Data" button |
| Dismiss | Ghost close button at right |
| Threshold | Show when storage exceeds 80% of plan limit |

### 8. EnrollmentBadge (payroll-dashboard)

**Purpose:** "+10" indicator next to enrollment fee amount.

| Element | Specification |
|---------|---------------|
| Layout | Inline `Badge` adjacent to enrollment fee dollar amount |
| Style | `warningBg` background, `warning` text color, fontSize 10px, fontWeight 700, borderRadius full |
| Content | "+10" |
| Condition | Only visible when enrollment qualifies for $124 enrollment bonus |

## Interaction Contracts

### Socket.IO Events (new)

| Event | Source | Consumer | UI Effect |
|-------|--------|----------|-----------|
| `alert:created` | CS chargeback submission | Payroll dashboard | New row appears in PayrollAlertTable with `HIGHLIGHT_GLOW` animation |
| `cs:changed` | CS chargeback/pending term CRUD | CS dashboard | Table rows refresh with highlight |
| `alert:resolved` | Payroll approve/clear action | Payroll dashboard | Row removed from alert table with fade-out |

### Toggle Behavior (PAY-01)

| State | Action | Guard | Result |
|-------|--------|-------|--------|
| Unpaid period | Click "Mark Paid" | None | Period marked paid, toggle text changes to "Mark Unpaid" |
| Paid period, OPEN | Click "Mark Unpaid" | Confirmation dialog | Period reverted to unpaid |
| Paid period, CLOSED | "Mark Unpaid" | Disabled, tooltip: "Cannot unpay a closed period" | No action |

### Sale Parser Flow (MGR-02 through MGR-06)

1. User pastes text into existing textarea
2. System calls `parseReceipt()` (fixed to map enum values correctly)
3. `ParsePreviewCard` appears with parsed key-value pairs and matched products
4. User reviews, clicks "Confirm & Fill" or "Discard"
5. On confirm: form fields populated, lead source at top, state field filled, matched products pre-selected
6. Core product NOT auto-selected unless explicitly parsed

### Permission Table Interaction (REP-04)

1. Table loads with role-level defaults pre-checked
2. Owner toggles individual checkboxes to create per-user overrides
3. Changed cells visually distinguish from defaults (dot indicator)
4. "Save Permissions" persists all overrides
5. Hard-coded rows (Payroll, User Creation) show lock icon, tooltips explain SUPER_ADMIN restriction
