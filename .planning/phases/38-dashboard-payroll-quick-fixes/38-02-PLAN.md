**Fix fallback extraction (lines 216-222):**

The fallback at lines 216-222 runs when `out.enrollmentFee` is not set by the primary path. After the primary fix, the fallback only runs when no enrollment line was found in product blocks. Update the fallback to also handle $0:

Change the condition at line 221 from:

to:

Actually, since the fallback block re-scans the full text with a global regex, it needs its own boolean. The simplest fix: since `enrollmentFound` from the primary path means `out.enrollmentFee` is already set, and the fallback only runs when `!out.enrollmentFee`, just add a local boolean:

```typescript
if (!out.enrollmentFee) {

  while ((efm = efRe.exec(t)) !== null) { ef += Number(efm[1].replace(/,/g, "")); efFound = true; }

}
```
