## Interaction Contracts

### 1. Paste-to-Parse (auto-parse on input)

| Step | Trigger | UI Response | Timing |
|------|---------|-------------|--------|
| User pastes/types into textarea | `onChange` event | Raw text appears in textarea | Instant |
| Parser runs | Same `onChange` handler | Consolidated records appear in preview table below textarea | Synchronous (no loading state needed) |
| Parse succeeds | Records found | Preview table fades in (`animate-fade-in`), Submit bar appears | 250ms fade |
| Parse finds malformed rows | Some rows cannot parse | Warning icon (AlertTriangle, lucide) with tooltip showing which fields failed, row still included with null fields | Instant |
| No valid rows | Empty or unparseable text | EmptyState remains visible, no preview table | Instant |

### 2. Inline Editing

| Field | Edit Mechanism | Validation |
|-------|---------------|------------|
| Posted Date | Native `<input type="date">` styled with `baseInputStyle` | Accept any valid date |
| Type | `Select` with options: ADVANCED COMM, OVERRIDE (extensible) | Required — default from parsed value |
| Member Company | Text `Input` inline in cell | Required — cannot be empty |
| Chargeback Amt | Text `Input` inline, accepts negative dollar format | Must be numeric, stored as negative |
| Total | Text `Input` inline, accepts dollar format | Must be numeric |
| Assigned To | `Select` populated from active rep roster | Nullable — can be unassigned |

### 3. Submit Flow

| Step | Trigger | UI Response |
|------|---------|-------------|
| Click "Submit Chargebacks" | Button click | Button shows loading state (spinner + "Submitting...") |
| API returns 201 | Success | Clear textarea, clear preview table, show success toast: "N chargebacks submitted" in green |
| API returns error | Failure | Show error toast: "Failed to submit (STATUS_CODE)" in red, preserve textarea + preview for retry |

### 4. Rep Roster Sidebar

| Action | Trigger | UI Response |
|--------|---------|-------------|
| Toggle sidebar | Click chevron icon | Sidebar slides in/out via `max-width` transition (250ms) |
| Add rep | Type name + click "Add" or press Enter | POST to API, rep appears in list with active toggle on |
| Toggle active/inactive | Click toggle switch | PATCH to API, toggle visual state, re-run round-robin on current preview records |
| Remove rep | Click X icon (visible on hover) | DELETE to API, rep removed from list, re-run round-robin |

### 5. Weekly Ticker

| Behavior | Detail |
|----------|--------|
| Initial load | GET /api/chargebacks/weekly-total on mount, display with AnimatedNumber count-up |
| After submit | Re-fetch weekly total, animate to new value |
| Display format | "$4,231.08 -- 12 records -- Week of 3/15-3/21" |
| No data | Show "$0.00 -- 0 records -- Week of M/D-M/D" |

## States Inventory

| State | Visual Treatment |
|-------|-----------------|
| Empty (no paste yet) | EmptyState component with clipboard icon, heading, and description |
| Parsing (instant) | No loading indicator needed -- parsing is synchronous client-side |
| Preview (records parsed) | Editable table with all columns, submit bar visible |
| Submitting | Primary button in loading state (spinner icon + "Submitting..."), all inputs disabled |
| Success | Textarea cleared, preview cleared, success toast (3s auto-dismiss), ticker re-fetches |
| Error (API) | Error toast (5s auto-dismiss), textarea and preview preserved for retry |
| Parse warnings | Individual cells with null values show "--" in muted text, row has warning icon in first column |
| Sidebar collapsed | Chevron points right, main content full width |
| Sidebar expanded | Chevron points left, sidebar visible at 240px |
| No active reps | Yellow warning text below roster: "No active reps -- records will be unassigned" |
