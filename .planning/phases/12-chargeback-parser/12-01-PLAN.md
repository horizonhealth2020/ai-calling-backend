**Route 1: POST /api/chargebacks** -- Submit consolidated chargeback records
- Auth: `requireAuth, requireRole("SUPER_ADMIN", "OWNER_VIEW")`
- Zod schema `chargebackSchema`:
  ```typescript
  const chargebackSchema = z.object({
    records: z.array(z.object({
      postedDate: z.string().nullable(),
      type: z.string().nullable(),
      payeeId: z.string().nullable(),
      payeeName: z.string().nullable(),
      payoutPercent: z.number().nullable(),
      chargebackAmount: z.number(),
      totalAmount: z.number().nullable(),
      transactionDescription: z.string().nullable(),
      product: z.string().nullable(),
      memberCompany: z.string().nullable(),
      memberId: z.string().nullable(),
      memberAgentCompany: z.string().nullable(),
      memberAgentId: z.string().nullable(),
      assignedTo: z.string().nullable(),
    })),
    rawPaste: z.string().min(1),
    batchId: z.string().min(1),
  });
  ```
- Use `prisma.chargebackSubmission.createMany` mapping each record to DB fields:
  - `postedDate: r.postedDate ? new Date(r.postedDate) : null`
  - `chargebackAmount: r.chargebackAmount` (Prisma accepts number for Decimal)
  - `totalAmount: r.totalAmount`
  - `payoutPercent: r.payoutPercent`
  - `submittedBy: (req as any).user.id`
  - `batchId`, `rawPaste`, `assignedTo` mapped directly
  - All other string fields mapped directly
- Response: `res.status(201).json({ count: result.count, batchId })`

**Route 2: GET /api/chargebacks/weekly-total** -- Weekly sum for ticker
- Auth: `requireAuth`
- Import and use `getSundayWeekRange` from `../services/payroll` (already imported at top of file)
- Calculate `weekStart` and `weekEnd` from `getSundayWeekRange(new Date())`
- Calculate `nextSunday` by adding 1 day to `weekEnd` for exclusive upper bound
- Use `prisma.chargebackSubmission.aggregate` with `_sum: { chargebackAmount: true }` and `_count: { id: true }` where `submittedAt: { gte: weekStart, lt: nextSunday }`
- Response: `res.json({ total: result._sum.chargebackAmount ?? 0, count: result._count.id, weekStart: weekStart.toISOString(), weekEnd: weekEnd.toISOString() })`

**Route 3: GET /api/cs-rep-roster** -- List all reps
- Auth: `requireAuth`
- On-access pruning: Before listing, delete inactive reps older than 30 days:
  ```typescript
  const thirtyDaysAgo = new Date();
  thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);
  await prisma.csRepRoster.deleteMany({
    where: { active: false, updatedAt: { lt: thirtyDaysAgo } },
  });
  ```
- Then `prisma.csRepRoster.findMany({ orderBy: { createdAt: "asc" } })`
- Response: `res.json(reps)`

**Route 4: POST /api/cs-rep-roster** -- Add rep
- Auth: `requireAuth`
- Zod: `z.object({ name: z.string().min(1).max(100) })`
- `prisma.csRepRoster.create({ data: { name } })`
- Response: `res.status(201).json(rep)`

**Route 5: PATCH /api/cs-rep-roster/:id** -- Toggle active
- Auth: `requireAuth`
- Zod: `z.object({ active: z.boolean() })`
- `prisma.csRepRoster.update({ where: { id: req.params.id }, data: { active } })`
- Response: `res.json(rep)`

**Route 6: DELETE /api/cs-rep-roster/:id** -- Remove rep
- Auth: `requireAuth`
- `prisma.csRepRoster.delete({ where: { id: req.params.id } })`
- Response: `res.status(204).end()`
