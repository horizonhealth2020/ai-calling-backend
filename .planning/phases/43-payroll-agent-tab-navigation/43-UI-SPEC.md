### Sidebar Row Structure

Each agent row (40px height):

```
| [dot?] Agent Name          $1,234  [badge] |
```

- Optional top-3 dot: 6px circle, `accentTeal`, positioned left (D-13)
- Agent name: 13px weight 500, `textSecondary` (or `textMuted` if zero sales)
- Earnings: 13px weight 400, `textTertiary`, right-aligned
- Status badge: inline, right-aligned after earnings (D-11)
- Horizontal padding: 16px (spacing.4)
- Vertical padding: 8px (spacing.2) top and bottom
- Hover: background transitions to `bgSurfaceOverlay`, duration 150ms
- Selected: background `bgSurfaceOverlay` + left border 3px solid `accentTeal`

### CS Section in Sidebar

- Divider: 1px solid `borderSubtle` with 8px vertical margin
- Section header: 11px uppercase, weight 700, `textTertiary`, letter-spacing 0.06em, text "CUSTOMER SERVICE"
- CS agent rows: identical styling to sales agent rows

## Interaction Contract

### Search Input

- Position: Top of sidebar, above agent list
- Style: `baseInputStyle` from tokens (14px, `bgSurfaceInset`, `borderDefault`)
- Placeholder: "Search agents..."
- Behavior: Filters agent list in real-time as user types (D-04)
- Clear: Typing clears previous filter; no explicit clear button needed
- Height: 36px (padding 8px 12px)

### Agent Selection

- Click agent row: replaces content area with selected agent's data (D-06)
- Visual feedback: selected row gets `bgSurfaceOverlay` background + 3px left teal border
- Transition on agent switch: instant content swap (no animation — design discretion)
- Socket.IO updates: must preserve selected agent via `useRef` guard (from STATE.md pitfall note)

### Load More Button

- Position: Bottom of content area, below last visible period
- Style: text-only button, `accentTeal` color, 13px weight 600
- Label: "Load More Periods"
- Hover: underline decoration
- Behavior: reveals all remaining periods from client-side data (D-09)
- Hidden when: all periods already visible, or fewer than 5 total periods
