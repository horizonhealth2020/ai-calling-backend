## Review Table Column Spec

Columns listed left to right. All use `baseThStyle` for headers, `baseTdStyle` for cells.

| # | Column | Width | Cell Content | Interactive |
|---|--------|-------|-------------|-------------|
| 1 | Status | 90px | `Badge` with match status text | No (read-only) |
| 2 | Member | 140px min | Member name + member ID on second line (smaller, `colors.textTertiary`) | No |
| 3 | Agent | 120px min | Agent name from matched sale; "--" if unmatched | No |
| 4 | Products | 200px min | Inline checkboxes: `[x] ProductName ($XX.XX)` per product | Yes (checkbox toggle) |
| 5 | Amount | 100px | `COMPACT_INPUT` type="text" showing formatted dollar value | Yes (always editable) |
| 6 | Rep | 130px | `<select>` dropdown with `COMPACT_INPUT` style, populated from CS reps | Yes (always editable) |
| 7 | Remove | 44px | `X` icon button | Yes (click to remove) |

### Column Behavior Notes

- **Products column (MATCHED):** All products start checked. Unchecking recalculates Amount to sum of remaining checked premiums. After user manually edits Amount, auto-recalculation stops for that row (set `amountManuallyOverridden: true`).
- **Products column (UNMATCHED):** Empty cell, no checkboxes. Amount stays as parsed.
- **Products column (MULTIPLE):** Shows a `<select>` dropdown listing candidate sale descriptions (agent name + member name). Once user selects a sale, products populate as checkboxes and status promotes to MATCHED. Format for dropdown options: `"AgentName - MemberName (N products)"`.
- **Status badge:** MATCHED rows show green badge, MULTIPLE shows yellow badge, UNMATCHED shows red badge. After user resolves a MULTIPLE by selecting a sale, badge changes to green MATCHED.

## Interaction States

### Loading State (Preview API Processing)

After user clicks "Parse & Preview" (modifies existing Parse button):
- Button shows `Loader2` icon spinning + "Matching..." text
- Button is disabled
- Textarea is disabled
- Duration: typically < 2 seconds for batches under 50 records

### Review Table Visible State

When `reviewRecords.length > 0`:
- Hide the textarea (keep rawText in state for error recovery)
- Show summary bar + review table + action buttons
- Action buttons below table: "Submit Batch" (primary, accent color) + "Clear" (secondary, ghost style)

### Row Removal + Undo

- Click X button: row disappears immediately from table
- Toast appears: `info` type, message: `"Removed {memberCompany || memberId || 'entry'}"`
- Toast duration: 5000ms (5 seconds)
- **Undo implementation note:** The existing `useToast()` does NOT support action buttons. Implementation must either: (a) extend ToastProvider to accept an `action` prop with `{ label: string, onClick: () => void }`, or (b) use a local "recently removed" state with a timed restore button rendered inline above the table. Option (a) is recommended -- add optional `action` prop to `ToastItem` interface and render an action button in `ToastEntry`.

### Submit Batch

- Button disabled when `reviewRecords.length === 0`
- Button disabled during submission (shows `Loader2` + "Submitting...")
- On success: clear all state, show success toast "N chargebacks submitted", emit `cs:changed` socket event once
- On error: show error toast with status code, keep review table intact so user can retry

### Empty State

When no paste data and no review records: existing textarea with placeholder is sufficient. No separate EmptyState component needed (matches current CSSubmissions pattern).

## Accessibility Notes

- Summary bar filter badges must be wrapped in `<button>` elements (Badge component renders `<span>`, which is not keyboard-focusable)
- Product checkboxes use native `<input type="checkbox">` with associated `<label>` for screen reader support
- Remove button must have `aria-label="Remove {memberCompany || memberId || 'entry'}"`
- MULTIPLE select must have `aria-label="Select matching sale for {memberCompany || memberId}"`
- Summary bar container should have `role="toolbar"` and `aria-label="Filter by match status"`
- Review table uses semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` elements (matches existing pattern)
