- Diff labels in edit confirmation: 14px, weight 400; diff values bolded at weight 700.
- Section sub-labels: 11px, `colors.textTertiary`, uppercase tracking — matches `baseLabelStyle`.

## Color

Source: `packages/ui/src/theme.css`, `packages/ui/src/tokens.ts`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#070a0a` (`--bg-root`) | Page background |
| Secondary (30%) | `#0c1414` (`--bg-surface`) / `#111c1c` (`--bg-surface-raised`) | Cards, sidebar, inline edit expansion rows, preview panel |
| Accent (10%) | `#14b8a6` (`--accent-teal`) / `linear-gradient(135deg, #14b8a6, #0d9488)` | Reserved elements listed below |
| Warning/Pending | `#f59e0b` (amber) | Pending edit request sections, pending badges, Clock icon |
| Destructive | `#f87171` (`--danger`) | Cancel edit request action, delete-class destructive buttons |
| Success | `#34d399` (`--success`) | Approve action button, "commission will increase" diff indicator |

Accent (`#14b8a6` / gradient) reserved for:
- Primary submit button (`SUBMIT_BTN`) — "Submit Sale"
- Commission total value in the preview panel (teal text color to signal it is a live calculated result)
- Border-focus ring on inputs (`--border-focus: #14b8a6`)
- Active nav tab indicator (existing pattern)

Accent is NOT used on: edit action buttons, cancel buttons, secondary labels, or any background fill wider than a single interactive element.

Pending/warning amber (`#f59e0b`) is reserved for:
- Inline pending edit badge on sale rows in the agent sales tab
- "Pending Approvals" section left-border (`3px solid #f59e0b`) and background (`rgba(245,158,11,0.08)`) — matches Phase 10 payroll pattern exactly
- Clock icon color within pending approval blocks

Shimmer loading state:
- Preview panel commission number during loading: `animation: "pulse 1.5s ease-in-out infinite"`, `opacity: 0.6` — reuses existing `.animate-pulse` keyframe from `packages/ui/src/animations.css`.
- Previous value remains visible (not replaced by spinner) to prevent layout shift.

## Interaction Contracts

### 1. Commission Preview Panel

**Location:** Top of the right column on the sales entry form, above the receipt parser / addon picker. The right column already exists in the two-column layout; preview occupies the topmost slot.

**Trigger:** Preview fires when any of these fields change: `productId`, `addonProductIds`, `addonPremiums`, `premium`, `enrollmentFee`, `paymentType`. Does NOT fire on: `memberName`, `memberId`, `carrier`, `notes`, `leadSourceId`, `saleDate`, `agentId`.

**Debounce:** 500ms on `premium` and `enrollmentFee` (typed numeric inputs). Immediate (no debounce) on dropdown changes (`productId`, `paymentType`, `addonProductIds`).

**Loading state:** Commission total number stays visible with `animation: "pulse 1.5s ease-in-out infinite", opacity: 0.6`. No spinner. No layout shift. Label changes to "Calculating..." (11px, textTertiary).

**Loaded state:** Total commission displayed in `PREVIEW_TOTAL` style (18px, teal, weight 700). Below it, a breakdown section shows:
- "Bundle Qualifier: Yes / No" — 14px, textSecondary
- Per-product line: `[Product Name] — $XX.XX` — shows full or half rate with reason label if halved
- "Bonus: +$10.00" if enrollment fee bonus applies — shown in `colors.success`
- "Period: Mar 22 – Mar 28" — formatted pay period for this sale

**Empty state (no product selected):** Panel shows "Select a product to see commission preview." in 14px textMuted. Panel is always rendered (not conditionally mounted) to hold its layout slot.

**Error state:** If preview API returns non-200: "Preview unavailable" in 12px `colors.danger`. Does not block form submission.

**Race condition guard:** Each preview call uses an `AbortController`. Previous in-flight request aborted when new trigger fires within the debounce window.

### 2. Inline Sale Editing — Manager Sales Tab

**Activation:** "Edit" (`Edit3` icon, 14px) button appears on every sale row in the agent sales tab. One row can be in edit mode at a time. Clicking Edit on a second row closes the first without saving.

**Edit mode visual:** Row background shifts to `colors.bgSurfaceRaised`. An expansion panel slides in below the row (`.animate-slide-down`, 250ms `ease-out`) containing all editable fields in a two-column grid.

**Pending sale guard:** If the sale has `hasPendingStatusChange: true` OR a pending `SaleEditRequest`, clicking Edit shows an inline message instead of the edit form: "A change is already pending for this sale. Wait for payroll to review before editing." (14px, `colors.warning`). Edit button remains visible but the expansion shows only this message.

**Field layout inside edit expansion:**
- Row 1: Product (dropdown, full width)
- Row 2: Premium | Enrollment Fee (two columns)
- Row 3: Payment Type | Agent (reassignment — two columns)
- Row 4: Addons (multi-select, same as entry form)
- Row 5: Carrier | Member Name | Member State (three columns)
- Row 6: Sale Date | Effective Date | Lead Source (three columns)
- Row 7: Notes (textarea, full width)

**Live diff panel:** As the manager modifies fields, a "Changes" summary renders below the edit fields. It shows only changed fields: `[Field]: [old value] → [new value]` with `DIFF_OLD` and `DIFF_NEW` styles. Commission and Period lines always show if financial fields changed, pulled from the preview endpoint.

**Save behavior:**
- MANAGER role: "Submit for Approval" button (`SUCCESS_BTN`). Creates `SaleEditRequest`. Row collapses. `PENDING_EDIT_BADGE` appears on the row.
- PAYROLL / SUPER_ADMIN role: "Save Changes" button (accent gradient, matches `SUBMIT_BTN` style). Applies directly. Row collapses. Toast/alert confirms save.

**Cancel:** `CANCEL_BTN` labeled "Discard Changes". Collapses edit expansion. No state persisted.

**Clearing edit state on tab switch:** `editingSaleId` resets to `null` when user switches away from the "Agent Sales" tab.

### 3. Payroll Dashboard — Pending Edit Requests

**Placement:** Within the existing amber "Pending Approvals" block per agent. Edit requests render alongside status change requests, distinguished by a badge type.

**Badge type distinction:**
- Status change: blue-tinted badge label "Status Change" (`colors.info` background at 10% opacity)
- Edit request: teal-tinted badge label "Edit Request" (`colors.accentTeal` background at 10% opacity)

**Edit request row layout:** Identical DOM structure to status change request rows. Shows:
- Member name + Member ID + Product name (14px, weight 700, `colors.textPrimary`)
- Requested by + date (11px, `colors.textMuted`)
- Field diff summary: each changed field on its own line, `[Field]: [old] → [new]` (11px, `colors.textSecondary`)
- Commission diff line if commission changes: "Commission: $45.00 → $90.00" (11px, success color for increase, danger for decrease)

**Approve / Reject buttons:** Same `BTN_ICON` style as Phase 10 status change buttons. Approve = `background: "#059669"`. Reject = `background: colors.dangerBg, border: "1px solid rgba(248,113,113,0.2)", color: colors.danger`.

**Confirmation for approval of finalized-period edits:** If the sale being edited is in a finalized period, approval shows `window.confirm`: "Approving this edit will create an adjustment entry in the next open period. Commission difference will be applied there. Continue?" — matches `window.confirm` pattern from Phase 10.

## Motion Contract

Source: `packages/ui/src/animations.css`, `packages/ui/src/tokens.ts` `motion` export.

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Edit expansion open | `.animate-slide-down` (slideDown keyframe) | 250ms (`motion.duration.normal`) | `cubic-bezier(0.16, 1, 0.3, 1)` (`motion.easing.out`) |
| Edit expansion close | Instant collapse (height 0, overflow hidden) | 150ms (`motion.duration.fast`) | `motion.easing.out` |

| Preview panel mount | `.animate-fade-in` | 250ms | `motion.easing.out` |
| Diff section appear | `.animate-fade-in-up` | 250ms | `motion.easing.out` |
| Approval row enter | `.animate-fade-in` | 250ms | `motion.easing.out` |
| Approval row exit (after action) | `opacity: 0` transition | 150ms | `motion.easing.out` |
| Stagger on edit field rows | `.stagger-1` through `.stagger-7` (50ms increments) | — | — |

Reduced motion: All animations respect `@media (prefers-reduced-motion: reduce)` via global rule in `animations.css` — no per-component handling needed.

## Accessibility Contract

- All icon-only buttons (`Edit3`, `Save`, `X`) must have `aria-label` attributes.
- Preview panel uses `aria-live="polite"` so screen readers announce commission updates without interrupting flow.
- Edit expansion uses `aria-expanded` on the edit button to communicate open/close state.
- Diff "Changes" section uses `role="status"` to announce field changes.
- Color alone is never the sole indicator: diff old values use `textDecoration: "line-through"` in addition to muted color; diff new values use weight 700 in addition to success color.
- Focus management: When edit expansion opens, first focusable field (Product dropdown) receives focus.
- Minimum touch target: 44px height on all interactive buttons.
