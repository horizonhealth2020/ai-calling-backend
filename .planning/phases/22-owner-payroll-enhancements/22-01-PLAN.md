**5. Add Socket.IO listener for service-payroll:changed event:**

After the existing `sale:changed` socket listener useEffect block (after line 420), add a new useEffect:

```tsx
// Socket.IO: refetch periods when service payroll changes
useEffect(() => {
  if (!socket) return;
  const handler = () => {
    authFetch(`${API}/api/reporting/periods?view=${periodView}`)
      .then(res => res.ok ? res.json() : { periods: [] })
      .then(data => setPeriods(data.periods ?? []))
      .catch(() => {});
  };
  socket.on("service-payroll:changed", handler);
  return () => { socket.off("service-payroll:changed", handler); };
}, [socket, API, periodView]);
```

This refetches the full period data rather than doing optimistic patching, because the csPayrollTotal is a server-computed aggregate.
