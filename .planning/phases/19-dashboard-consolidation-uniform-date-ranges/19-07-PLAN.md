For each sub-tab component that displays KPI counters, add the DateRangeFilter and wire it to the API calls:

**Pattern for each component:**
1. Import `useDateRange` from `@/lib/DateRangeContext`
2. Import `DateRangeFilter, KPI_PRESETS` from `@ops/ui`
3. Get date range from context: `const { value: dateRange, onChange: setDateRange } = useDateRange();`
4. Render `<DateRangeFilter value={dateRange} onChange={setDateRange} presets={KPI_PRESETS} />` ABOVE the KPI cards section
5. Build query params from dateRange value:
   ```typescript
   function buildDateParams(dr: DateRangeFilterValue): string {
     if (dr.preset === "custom" && dr.from && dr.to) return `&from=${dr.from}&to=${dr.to}`;
     if (dr.preset && dr.preset !== "custom") return `&range=${dr.preset}`;
     return "";
   }
   ```
6. Append date params to ALL fetch calls that load KPI data
7. Add `dateRange` to the dependency array of useEffect/useCallback that fetches KPI data, so data re-fetches when range changes
