  <behavior>
    COMM-01: Core (premium=100, commissionAbove=50%, threshold=50) + Compass VAB addon (isBundleQualifier=true, premium=10)
      -> bundlePremium = 100 (VAB excluded), rate = 50% (100 >= 50 threshold)
      -> commission = 100 * 0.50 = 50.00

    COMM-02: Core (premium=100, commissionAbove=50%, threshold=50) alone, no Compass VAB
      -> bundlePremium = 100, rate = 50%
      -> pre-halving = 50.00, halved because no qualifier
      -> commission = 25.00

    COMM-02 + commissionApproved override: Same as above but commissionApproved=true
      -> commission = 50.00 (halving skipped)

    COMM-04: Core (premium=100, commissionAbove=50%, threshold=150) + regular addon (premium=60) + Compass VAB
      -> bundlePremium = 100 + 60 = 160, rate = 50% (160 >= 150 threshold)
      -> commission = 160 * 0.50 = 80.00

    COMM-04 below threshold: Core (premium=100, commissionBelow=25%, threshold=200) + regular addon (premium=60) + Compass VAB
      -> bundlePremium = 100 + 60 = 160, rate = 25% (160 < 200 threshold)
      -> commission = 160 * 0.25 = 40.00

    COMM-05: Standalone addon (premium=80, standaloneCommission=30%)
      -> commission = 80 * 0.30 = 24.00

    COMM-05 with null standaloneCommission: Standalone addon (premium=80, standaloneCommission=null)
      -> commission = 0.00

    COMM-06: Standalone AD&D (premium=50, standaloneCommission=35%)
      -> commission = 50 * 0.35 = 17.50

    COMM-06 with null: Standalone AD&D (premium=50, standaloneCommission=null)
      -> commission = 0.00

    COMM-07: Core + AD&D (premium=50, bundledCommission=70%) + Compass VAB
      -> AD&D commission = 50 * 0.70 = 35.00
      -> total = core commission + 35.00

    COMM-07 halving: Core + AD&D (premium=50, bundledCommission=70%), NO Compass VAB
      -> pre-halving total = core commission + 35.00
      -> halved total (both core bundle and AD&D halved)

    COMM-11: Core (premium=33.33, commissionAbove=50%, threshold=0) + Compass VAB
      -> commission = 33.33 * 0.50 = 16.665 -> rounded to 16.67

    Addon premium sourcing: Each addon uses its own SaleAddon.premium, NOT sale.premium

**GREEN phase:** Rewrite `calculateCommission()` and remove `calcProductCommission()`:

1. Delete the `calcProductCommission()` helper function entirely.

2. Rewrite `calculateCommission()` following this algorithm:
   a. Build product entries from sale: `{ product, premium }` for primary product (premium from sale.premium) and each addon (premium from addon.premium ?? 0)
   b. Classify: find coreEntry, check qualifierExists (any product with isBundleQualifier), filter AD&D entries, filter regular addon entries (type ADDON and NOT isBundleQualifier)
   c. If core exists:
      - Sum bundle premium = core premium + regular addon premiums (EXCLUDE qualifier premiums)
      - Determine rate from core product: bundlePremium >= threshold ? commissionAbove : commissionBelow (null rates = 0)
      - Bundle commission = bundlePremium * (rate / 100)
      - AD&D commission: for each AD&D entry, add entry.premium * (bundledCommission / 100); null bundledCommission = 0
      - Total = bundle commission + AD&D commission
      - If no qualifier exists AND not commissionApproved: halve total
   d. If no core (standalone):
      - For each AD&D: premium * (standaloneCommission / 100); null = 0
      - For each ADDON: premium * (standaloneCommission / 100); null = 0
      - Total = sum of all standalone commissions
   e. Apply enrollment fee rules via existing `applyEnrollmentFee()` -- DO NOT modify this function
   f. Round final result: `Math.round((finalCommission + enrollmentBonus) * 100) / 100`

3. Remove:
   - String matching: `n.includes("compass") && n.includes("vab")` -- replaced by `isBundleQualifier` check
   - FL exemption: `sale.memberState?.toUpperCase() === "FL"` -- removed entirely per user decision
   - Hardcoded AD&D fallback rates: `?? 70` and `?? 35` -- null means $0
   - Hardcoded addon fallback rate: `?? 30` -- null means $0

4. Keep `applyEnrollmentFee()` EXACTLY as-is (Phase 3 scope).
5. Keep `upsertPayrollEntryForSale()` EXACTLY as-is (it calls calculateCommission and uses the returned number).
6. Keep `getSundayWeekRange()` EXACTLY as-is.
7. Keep the `SaleWithProduct` type EXACTLY as-is.

8. Log warnings when null commission rates are encountered (console.warn with product ID and rate field name) -- helps ops team detect misconfigured products.

Run `npm run test:ops` -- all tests should PASS.
