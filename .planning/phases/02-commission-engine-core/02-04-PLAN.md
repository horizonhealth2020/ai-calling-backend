In the POST /sales route (around line 279-319):

1. Add `addonPremiums` to the Zod schema:
   ```
   addonPremiums: z.record(z.string(), z.number().min(0)).default({})
   ```
   This accepts a map of productId -> premium amount.

2. Update the addon creation in the Prisma create call. Change:
   ```
   addons: { create: addonProductIds.map(id => ({ productId: id })) }
   ```
   To:
   ```
   addons: { create: addonProductIds.map(id => ({ productId: id, premium: addonPremiums[id] ?? null })) }
   ```
   Extract `addonPremiums` from `parsed.data` alongside `addonProductIds`.

This is a 2-line change to the API. The premium field already exists on SaleAddon (Decimal(12,2), optional), so no schema migration needed.
