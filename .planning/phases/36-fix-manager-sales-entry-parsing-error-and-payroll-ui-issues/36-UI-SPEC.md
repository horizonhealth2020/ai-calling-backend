## Interaction Contracts

### Fix 1: matchProduct() — No UI Changes

### Fix 3: ACA PL Entry — New UI Elements

#### 3a. ACA Checkbox on Existing Sale Form

**Location:** Right side of the sale entry form, below the addons section, above the submit button.

**Layout:**
```
[  Include ACA Plan  ] ---- checkbox row, right-aligned in form
    |
    v  (conditional, shown when checked)
[  Carrier         ] [ Member Count ]
   (text input)       (number input, min=1, default=1)
```

**Checkbox row style:**
```typescript
const ACA_TOGGLE_ROW: React.CSSProperties = {
  display: "flex",
  alignItems: "center",
  gap: spacing[2],            // 8px
  padding: `${spacing[2]}px 0`,
  marginTop: spacing[3],      // 12px
};
```

**Checkbox label style:**
```typescript
const ACA_TOGGLE_LABEL: React.CSSProperties = {
  fontSize: 13,
  fontWeight: 600,
  color: colors.textPrimary,
  cursor: "pointer",
  display: "flex",
  alignItems: "center",
  gap: 6,
};
```

**Conditional fields container:**
```typescript
const ACA_FIELDS: React.CSSProperties = {
  display: "grid",
  gridTemplateColumns: "1fr 120px",
  gap: 14,
  marginTop: spacing[2],      // 8px
  padding: `${spacing[3]}px`,  // 12px
  background: colors.bgSurfaceInset,
  borderRadius: radius.lg,     // 8px
  border: `1px solid ${colors.borderSubtle}`,
};
```

**Interaction:**
- Checkbox unchecked by default
- When checked: carrier input and member count input slide into view (no animation, instant show)
- Carrier input: free-text, placeholder "Enter carrier name"
- Member count input: number type, min 1, default value "1", placeholder "1"
- On form submit with ACA checked: creates regular sale first, then ACA_PL sale linked to it
- Validation: if ACA checked, carrier is required (non-empty). Member count defaults to 1 if empty.

#### 3b. ACA Standalone Entry Section

**Location:** Below the main sale entry form, separated by a horizontal rule.

**Layout:**
```
--- horizontal separator ---

ACA-Only Entry                    [v] (collapsible toggle)
  |
  v  (shown when expanded)
[ Agent       ] [ Member Name  ]     -- row 1, 2-col grid
[ Carrier     ] [ Member Count ]     -- row 2, 2-col grid
              [ Submit ACA Entry ]   -- right-aligned button
```

**Section header style:**
```typescript
const ACA_SECTION_HEADER: React.CSSProperties = {
  fontSize: 16,
  fontWeight: 600,
  color: colors.textPrimary,
  display: "flex",
  alignItems: "center",
  justifyContent: "space-between",
  cursor: "pointer",
  padding: `${spacing[3]}px 0`,  // 12px vertical
};
```

**Section container style:**
```typescript
const ACA_SECTION: React.CSSProperties = {
  marginTop: spacing[4],          // 16px
  paddingTop: spacing[4],         // 16px
  borderTop: `1px solid ${colors.borderSubtle}`,
};
```

**Form grid style:**
```typescript
const ACA_FORM_GRID: React.CSSProperties = {
  display: "grid",
  gridTemplateColumns: "1fr 1fr",
  gap: 14,
};
```

**Interaction:**
- Section collapsed by default (only header visible with chevron icon)
- Click header to expand/collapse
- Agent field: Select dropdown using existing `<Select>` component
- Member Name: Input using existing `<Input>` component, placeholder "Member full name"
- Carrier: Input, placeholder "Enter carrier name"
- Member Count: number input, min 1, default "1"
- Submit button: uses existing Button component, label "Submit ACA Entry", primary variant
- On submit: creates ACA_PL sale with no premium, no lead source
- Validation: agent required, member name required, carrier required, member count >= 1

#### 3c. ACA Badge in Payroll Agent Pay Cards

**Location:** Inline with the product name in each payroll entry row within agent pay cards.

**Badge style:**
```typescript
const ACA_BADGE: React.CSSProperties = {
  display: "inline-flex",
  alignItems: "center",
  fontSize: 11,
  fontWeight: 700,
  color: colors.info,
  background: colors.infoBg,
  padding: "2px 8px",
  borderRadius: radius.full,
  marginLeft: spacing[2],        // 8px
  letterSpacing: typography.tracking.wide,
  textTransform: "uppercase",
};
```

**Content:** Text "ACA" displayed after the product name.

**Commission display:** Shows flat commission as "$X.XX x N members = $total" format instead of percentage.

```typescript
// Commission cell text for ACA entries
const acaCommissionText = `$${flatAmount.toFixed(2)} x ${memberCount} = ${formatDollar(totalCommission)}`;
```
