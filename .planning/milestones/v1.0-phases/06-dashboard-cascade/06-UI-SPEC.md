**Exception — 12px and 20px:** These values fall outside the standard set {4, 8, 16, 24, 32, 48, 64} but are inherited production tokens defined in `packages/ui/src/tokens.ts` (lines 56, 58) as `spacing[3] = 12` and `spacing[5] = 20`. Changing them would break visual consistency with the established design system. Phase 6 does not introduce these values -- it inherits them.

## Typography

Inherited from `packages/ui/src/tokens.ts`. Phase 6 introduces one new text element (disconnection banner, no new headings or display text). This phase uses exactly 2 font weights:

| Role | Size | Weight | Line Height | Phase 6 Usage |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 | 1.6 | Not new -- existing dashboard body text |
| Label | 11px | 700 | 1.45 | Not new -- existing label style |
| Banner text | 13px | 700 | 1.5 | Disconnection banner message |
| Heading | 18px | 700 | 1.4 | Not new -- existing page titles |

**Declared weights for this phase:** 400 (regular) and 700 (bold). Banner text uses `typography.weights.bold` (700) to match the existing label and heading weight, providing high-urgency visual emphasis appropriate for a disconnection warning. The `baseButtonStyle` in tokens.ts uses weight 600 for buttons, but Phase 6 introduces no new buttons -- the banner is the only new text element and uses 700.

## Color

Inherited from `packages/ui/src/theme.css`. Phase 6 introduces two new color applications:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg-root)` / #070a0a dark, #f5f7f7 light | Background, surfaces -- no change |
| Secondary (30%) | `var(--bg-surface)` / #0c1414 dark, #ffffff light | Cards, sidebar, nav -- no change |
| Accent (10%) | `var(--accent-teal)` / #14b8a6 | Reserved for: primary CTAs, active nav indicators, focus rings -- no change |
| Highlight glow | `rgba(59, 130, 246, 0.15)` (blue) | New sale/update row highlight animation |
| Disconnection | `var(--danger)` / #f87171 at 90% opacity | Disconnection banner background |

### Highlight Glow Specification

The highlight glow uses **blue** (`rgba(59, 130, 246, 0.15)`) rather than the teal accent. This is a deliberate choice from CONTEXT.md to differentiate "something just changed" (blue glow) from "interactive element" (teal accent). The blue is informational, not actionable.

| Property | Value |
|----------|-------|
| Glow color | `rgba(59, 130, 246, 0.15)` |
| Secondary glow | `rgba(59, 130, 246, 0.08)` |
| Box shadow active | `0 0 20px rgba(59, 130, 246, 0.15), 0 0 40px rgba(59, 130, 246, 0.08)` |
| Box shadow resting | `none` (or element's default shadow) |
| Transition property | `box-shadow` |
| Transition duration | 1500ms |
| Transition easing | `ease-out` |

**Two-phase animation approach:** Apply the highlight box-shadow immediately via state, then after one animation frame (`requestAnimationFrame`), remove the highlight state. The CSS `transition: box-shadow 1.5s ease-out` on the element handles the smooth fade. This avoids the snap-off problem documented in RESEARCH.md Pitfall 7.

Implementation pattern:
1. Element always has `transition: box-shadow 1.5s ease-out` in its base style
2. On event: add sale ID to `highlightedIds` Set (box-shadow applied)
3. After 100ms (`setTimeout`): remove sale ID from `highlightedIds` Set
4. CSS transition handles the 1.5s fade-out automatically

### Disconnection Banner Color

| Property | Value |
|----------|-------|
| Background | `rgba(239, 68, 68, 0.9)` (#ef4444 at 90% opacity) |
| Text color | `#ffffff` |
| Backdrop filter | `blur(8px)` |

**Source:** RESEARCH.md disconnection banner example, using red-500 for urgency. Consistent with existing `var(--danger)` semantic color family.

Accent reserved for: primary CTA buttons, active sidebar nav indicators, focus rings, link hover states. Never for highlight animations or status banners.

## Component Inventory

Phase 6 adds exactly 2 new visual elements. No new pages, forms, or interactive controls.

### 1. Disconnection Banner

Fixed-position banner at top of viewport. Appears after 10 seconds of Socket.IO disconnection. Disappears immediately on reconnect.

| Property | Value |
|----------|-------|
| Position | `fixed`, top: 0, left: 0, right: 0 |
| z-index | 9999 |
| Padding | `8px 16px` (spacing.2 vertical, spacing.4 horizontal) |
| Background | `rgba(239, 68, 68, 0.9)` |
| Text color | `#ffffff` |
| Font size | 13px (`typography.sizes.sm`) |
| Font weight | 700 (`typography.weights.bold`) |
| Text align | center |
| Backdrop filter | `blur(8px)` |
| Copy | "Connection lost. Reconnecting..." |

### 2. Row/Card Highlight Glow

Applied to existing table rows (manager tracker, payroll entries) and cards (owner KPI StatCards) when a real-time update arrives. Not a standalone component -- it is a style modifier applied to existing elements.

| Property | Value |
|----------|-------|
| Applied to | Table rows (manager, payroll), StatCards (owner), leaderboard rows (sales board) |
| Trigger | `sale:changed` Socket.IO event |
| Active state | `boxShadow: "0 0 20px rgba(59, 130, 246, 0.15), 0 0 40px rgba(59, 130, 246, 0.08)"` |
| Resting state | Element's default box-shadow (or `none`) |
| Duration | 1500ms fade-out via CSS transition |
| Base style addition | `transition: "box-shadow 1.5s ease-out"` added to highlightable elements |

## Interaction Contracts

### Real-Time Update Behavior Per Dashboard

| Dashboard | Element Updated | On `created` | On `status_changed` (Dead/Declined to Ran) |
|-----------|----------------|-------------|---------------------------------------------|
| Manager (agent tracker) | Sale row in agent's sales list | Insert new row at top + highlight + increment agent sale count | Insert row (was invisible per STATUS-08) + highlight |
| Sales board | Leaderboard rankings | Re-sort agent rankings, highlight changed agent row | Treat as new sale appearing, re-sort, highlight |
| Payroll | Agent payroll card entries | Insert entry in matching agent's period card + highlight | Insert entry + highlight |
| Owner | KPI StatCards | Swap number instantly (no count animation) + highlight card | Swap number + highlight card |

### Owner KPI Update

Per CONTEXT.md: numbers swap instantly with no count-up animation. The `AnimatedNumber` component in `@ops/ui` is NOT used for real-time event updates. The highlight glow on the StatCard provides the visual cue that something changed.

### Sales Board Re-Sort

Per CONTEXT.md: leaderboard rankings re-sort on every event. If an agent jumps position, the new position row gets the highlight glow. The board should feel dynamic and competitive.

### Reconnect Behavior

| Event | Timing | Action |
|-------|--------|--------|
| Socket disconnects | Immediately | Start 10-second timer |
| Still disconnected at 10s | 10s after disconnect | Show disconnection banner |
| Socket reconnects | On reconnect event | Hide banner, trigger full data refetch for each dashboard |

## Phase-Specific Design Notes

### What This Phase Does NOT Change

- No new pages or routes
- No new forms or interactive controls
- No changes to existing layout, typography, or color tokens
- No changes to the `@ops/ui` PageShell or any shared components
- No new icons or illustrations

### What This Phase Adds

1. A `useSocket` hook in `@ops/socket` that returns `{ disconnected: boolean }`
2. A disconnection banner rendered conditionally in each dashboard's main page when `disconnected === true`
3. A highlight glow effect applied to existing rows/cards via a `highlightedIds` state Set
4. State patching logic in each dashboard's `onSaleChanged` callback (no visual changes beyond the highlight)

### Style Constants to Create

Each dashboard page that integrates Socket.IO should add these two style constants:

```typescript
const DISCONNECT_BANNER: React.CSSProperties = {
  position: "fixed",
  top: 0,
  left: 0,
  right: 0,
  padding: "8px 16px",
  background: "rgba(239, 68, 68, 0.9)",
  color: "#fff",
  textAlign: "center",
  fontSize: 13,
  fontWeight: 700,
  zIndex: 9999,
  backdropFilter: "blur(8px)",
};

const HIGHLIGHT_GLOW: React.CSSProperties = {
  boxShadow: "0 0 20px rgba(59, 130, 246, 0.15), 0 0 40px rgba(59, 130, 246, 0.08)",
};
```

The `DISCONNECT_BANNER` constant can be defined once in `@ops/socket` and exported, or defined per-dashboard. The `HIGHLIGHT_GLOW` style is merged into existing row/card styles when the item's ID is in the `highlightedIds` Set.
