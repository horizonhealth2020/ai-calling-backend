1. Add `isBundleQualifier Boolean @default(false) @map("is_bundle_qualifier")` to the Product model in `prisma/schema.prisma`, placed after `standaloneCommission` and before `enrollFeeThreshold`.

2. Create a manual migration directory under `prisma/migrations/` named `20260315000000_add_bundle_qualifier/` with a `migration.sql` file containing:

```sql
-- Add isBundleQualifier column to products
ALTER TABLE "products" ADD COLUMN "is_bundle_qualifier" BOOLEAN NOT NULL DEFAULT false;

-- Flag existing Compass VAB product(s) as bundle qualifiers
UPDATE "products" SET "is_bundle_qualifier" = true
WHERE LOWER("name") LIKE '%compass%' AND LOWER("name") LIKE '%vab%';
```

3. Run `npx prisma generate` to regenerate the Prisma client types (so `product.isBundleQualifier` is available in TypeScript).

NOTE: Do NOT run `prisma migrate dev` since there is no DATABASE_URL in the dev environment (per Phase 1 decision -- manual migration SQL). The migration SQL file serves as documentation and will be applied via `prisma migrate deploy` in deployed environments.
