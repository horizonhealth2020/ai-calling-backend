## Spacing Scale

Declared values from `@ops/ui` tokens (multiples of 4):

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| spacing[1] | 4px | Icon gaps within badge |
| spacing[2] | 8px | Gap between label and value in view mode |
| spacing[3] | 12px | Compact card internal padding |
| spacing[4] | 16px | Gap between form fields in edit mode |
| spacing[6] | 24px | Card padding (matches `baseCardStyle`) |
| spacing[8] | 32px | Gap between ACA PL group section and previous group |

Exceptions: spacing[3] (12px) is not in the strict 8-point scale but IS a defined `@ops/ui` design token (`--space-3: 12px` in `packages/ui/src/theme.css` line 46). It is used extensively across existing components: Button sm/lg padding, Toast padding, TabNav tab padding, and responsive card padding. Retaining 12px maintains consistency with these established patterns. Using 8px would be too tight for card internal padding; 16px would duplicate the form field gap and reduce visual hierarchy between compact and standard spacing.

## Typography

All values reference existing `@ops/ui` typography tokens. No new sizes introduced. Two weights only: 400 (normal) and 600 (semibold).

| Role | Size | Weight | Line Height | Token Path | Notes |
|------|------|--------|-------------|------------|-------|
| Group heading | 14px | 600 (semibold) | 1.6 | `typography.sizes.base` + `typography.weights.semibold` | Uppercase + `letterSpacing: tracking.caps` provides sufficient hierarchy without bold |
| Card product name | 16px | 600 (semibold) | 1.5 | `typography.sizes.md` + `typography.weights.semibold` | |
| Field label | 11px | 600 (semibold) | 1.45 | `baseLabelStyle` (uppercase, tracking.caps) | Uppercase + small size + semibold distinguishes labels without requiring bold |
| Field value | 14px | 400 (normal) | 1.6 | `typography.sizes.base` + `typography.weights.normal` | |
| Commission amount | 14px | 600 (semibold) | 1.6 | `typography.sizes.base` + `typography.weights.semibold` | |
| Input text | 14px | 400 (normal) | 1.6 | `baseInputStyle` | |

## Layout Specification

### ACA PL Group Section

```
+------------------------------------------------------------------+
| [info-blue left border 3px] ACA PL                    (2 products)|
+------------------------------------------------------------------+
| [ProductCard: ACA PL product 1]                                   |
| [ProductCard: ACA PL product 2]                                   |
+------------------------------------------------------------------+
```

- Group placed LAST in the Products tab, after CORE, Add-on, and AD&D groups
- Group heading bar: `borderLeft: 3px solid colors.info`, matching existing group heading pattern
- Product count shown as a small Badge in the heading bar
- Cards rendered in a vertical stack with `spacing[4]` (16px) gap between cards

### ACA PL Card — View Mode

```
+------------------------------------------------------------------+
| [ACA PL badge]  Product Name                        [Edit button] |
|                                                                    |
| FLAT COMMISSION                                                    |
| $25.00 per member                                                 |
|                                                                    |
| NOTES (if present)                                                 |
| Optional note text here                                           |
+------------------------------------------------------------------+
```

- Same card chrome as existing CORE/ADDON/AD_D cards: `bgSurface` background, `borderDefault` border, `radius.xl` corners
- No premium threshold row, no percentage commission rows, no bundled/standalone rows
- Flat commission displayed as: `$` + value formatted to 2 decimals + ` per member` suffix
- If `flatCommission` is null/undefined: show "Not configured" in `textTertiary`

### ACA PL Card — Edit Mode

```
+------------------------------------------------------------------+
| [ACA PL badge]  [Name input]                  [Save] [Cancel]    |
|                                                                    |
| FLAT COMMISSION ($ PER MEMBER)                                     |
| [number input: step=0.01, min=0, placeholder="e.g. 25.00"]       |
|                                                                    |
| NOTES                                                              |
| [textarea input]                                                   |
+------------------------------------------------------------------+
```

- Type selector: hidden for ACA_PL products (type is not changeable per D-03)
- Only 2 editable fields: flatCommission (number) and notes (textarea)
- Name is editable (text input) consistent with other product types
- Active toggle shown consistent with other product types
- Save button: `colors.primary400` background, white text, 600 weight
- Cancel button: ghost style, `colors.textSecondary`

## Interaction States

| Interaction | Behavior |
|-------------|----------|
| Card hover | `boxShadow: shadows.md`, subtle lift (existing pattern) |
| Edit button click | Toggles card to edit mode, fields become inputs |
| Save click | Shows saving spinner, calls PATCH `/products/:id` with `{ flatCommission, name, notes, active }`, reverts to view mode on success |
| Save error | Toast notification with error message from API response |
| Cancel click | Reverts to view mode, discards unsaved changes |
| Empty flat commission input on save | Sends `flatCommission: null` to API |
| Input focus | Border transitions to `colors.borderFocus` (existing `input-focus` CSS class) |
