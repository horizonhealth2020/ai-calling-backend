```sql
-- Data migration: aggregate entry-level values to agent+period level
INSERT INTO agent_period_adjustments (id, agent_id, payroll_period_id, bonus_amount, fronted_amount, hold_amount, created_at, updated_at)
SELECT
  gen_random_uuid(),
  agent_id,
  payroll_period_id,
  SUM(bonus_amount),
  SUM(fronted_amount),
  SUM(COALESCE(hold_amount, 0)),
  NOW(),
  NOW()
FROM payroll_entries
WHERE bonus_amount > 0 OR fronted_amount > 0 OR COALESCE(hold_amount, 0) > 0
GROUP BY agent_id, payroll_period_id
ON CONFLICT (agent_id, payroll_period_id) DO UPDATE SET
  bonus_amount = EXCLUDED.bonus_amount,
  fronted_amount = EXCLUDED.fronted_amount,
  hold_amount = EXCLUDED.hold_amount;

-- Zero out entry-level fields after migration
UPDATE payroll_entries SET bonus_amount = 0, fronted_amount = 0, hold_amount = 0
WHERE bonus_amount > 0 OR fronted_amount > 0 OR COALESCE(hold_amount, 0) > 0;
```

**B. halvingReason preservation (FIX-08 root cause):**

In `apps/ops-api/src/services/payroll.ts`, the `calculateCommission` function returns `halvingReason: null` when `commissionApproved` is true because:
- Line 177: `if (bundleCtx && !bundleCtx.requiredAddonAvailable && !bundleCtx.fallbackAddonAvailable && !sale.commissionApproved)` — skips setting halvingReason when approved
- `applyEnrollmentFee` similarly skips feeHalvingReason when approved

Fix: Change `calculateCommission` to ALWAYS return the halvingReason regardless of `commissionApproved`. The halving (dividing commission by 2) should still be skipped when approved, but the REASON should always be returned.

Specifically in `calculateCommission`:
1. Line 177: Remove `&& !sale.commissionApproved` from the condition. Change:
   ```typescript
   if (bundleCtx && !bundleCtx.requiredAddonAvailable && !bundleCtx.fallbackAddonAvailable && !sale.commissionApproved) {
     halvingReason = bundleCtx.halvingReason;
   }
   ```
   To:
   ```typescript
   if (bundleCtx && !bundleCtx.requiredAddonAvailable && !bundleCtx.fallbackAddonAvailable) {
     halvingReason = bundleCtx.halvingReason;
   }
   ```

2. In `applyEnrollmentFee` function, ensure `feeHalvingReason` is returned regardless of `commissionApproved`. Find the condition that checks `commissionApproved` and remove it from the reason-setting logic. The actual halving (dividing by 2) should still check `commissionApproved` to skip halving.

3. At lines 203-205, change the halving logic to also check `commissionApproved`:
   ```typescript
   const reasons = [halvingReason, feeHalvingReason].filter(Boolean);
   const combinedReason = reasons.length > 0 ? reasons.join("; ") : null;
   const halvedCommission = (combinedReason && !sale.commissionApproved) ? finalCommission / 2 : finalCommission;
   ```
   This ensures: reasons are ALWAYS collected, but halving is ONLY applied when NOT approved.
