    Audit every exported symbol across shared packages and app-internal modules. Remove exports that have zero consumers in the entire codebase.

    APPROACH -- Audit shared packages first (highest risk), then app-internal code:

    **Step 1: Shared package exports (@ops/ui, @ops/socket, @ops/auth, @ops/types, @ops/utils, @ops/db)**

    For EACH export in each shared package:
    1. Identify the exported name (e.g., `Badge`, `useSocket`, `signSessionToken`)
    2. Search the ENTIRE codebase (apps/ AND packages/) for usage:
       ```
       grep -rn "EXPORT_NAME" apps/ packages/ --include="*.ts" --include="*.tsx" --include="*.js" | grep -v node_modules | grep -v ".next" | grep -v "export.*EXPORT_NAME"
       ```
    3. If zero results (excluding the export declaration itself), the export is dead -- remove it.
    4. If the export is a component in @ops/ui with zero consumers:
       - Remove the component file from packages/ui/src/components/
       - Remove the re-export from packages/ui/src/index.tsx
    5. If the export is from @ops/types, also search for type usage patterns: `: EXPORT_NAME`, `as EXPORT_NAME`, `extends EXPORT_NAME`, `import type { EXPORT_NAME }`.

    CRITICAL SAFETY:
    - Search ALL of apps/ AND packages/ -- not just one app. A shared package export may be used by any app.
    - Include .js files in the search (Morgan uses CommonJS)
    - Do NOT remove exports that are part of the public API contract of a package even if only used in tests
    - When removing a component from @ops/ui, also check if it's used in @ops/ui itself (e.g., Tooltip used inside another component)

    **Step 2: App-internal unreferenced functions**

    For ops-api services (apps/ops-api/src/services/*.ts):
    1. List every exported function
    2. Search apps/ops-api/src/ for call sites (excluding the declaration)
    3. Remove functions with zero call sites

    For ops-api route files: These are registered via router.use() in routes/index.ts, so route handler functions are "used" even if not directly called elsewhere. Only remove helper functions within route files that are never called.

    For Morgan (apps/morgan/*.js):
    1. List every module.exports and exported function
    2. Search apps/morgan/ for require() and usage of each export
    3. Remove functions with zero call sites
    4. Be careful with dynamic patterns -- read each file fully (only 7 files, ~1500 LOC)
