# Phase 28 Plan 01: Eliminate any types and add explicit type annotations

## Objective

Replace all explicit `any` type annotations in application code with proper types. Add explicit type annotations to shared package exports. Ensure API response types match actual response shapes.

## Context

- @apps/ops-api/src/ — Express REST API with ~50 `any` usages
- @apps/ops-dashboard/ — Next.js dashboard with ~96 `any` usages (mostly CSTracking and payroll)
- @packages/ — Shared packages with some `any` in auth/client and socket
- Requirements: TS-01 (no any), TS-02 (response type alignment), TS-03 (package export annotations)

## Tasks

### Task 1: Add proper types to shared packages (TS-03)
type="auto"

Replace `any` types in `packages/auth/src/client.ts`, `packages/socket/src/useSocket.ts`, and add explicit return type annotations to all shared package exports that lack them.

**Done when:** Every export from @ops/auth, @ops/types, @ops/utils, @ops/ui, @ops/db, and @ops/socket has an explicit type annotation. No `any` in package source files.

### Task 2: Replace any types in ops-api routes (TS-01, TS-02)
type="auto"

Replace all `any` type annotations in `apps/ops-api/src/routes/` with proper interfaces and types. Create shared type definitions where patterns repeat (e.g., addon premium reduction, Prisma query results).

**Done when:** Zero `any` annotations in ops-api route files.

### Task 3: Replace any types in ops-api services and workers (TS-01)
type="auto"

Replace all `any` type annotations in `apps/ops-api/src/services/`, `apps/ops-api/src/workers/`, `apps/ops-api/src/socket.ts`, and `apps/ops-api/src/index.ts`.

**Done when:** Zero `any` annotations in ops-api service, worker, and root files (excluding test files).

### Task 4: Replace any types in ops-dashboard (TS-01)
type="auto"

Replace all `any` type annotations in `apps/ops-dashboard/` source files with proper interfaces and types. Focus on CSTracking (chargebacks/pending terms), payroll pages, manager pages, and API routes.

**Done when:** Zero `any` annotations in ops-dashboard source files.

## Verification

```bash
# Should return zero results (excluding node_modules, .next, test mocks, .d.ts files)
grep -rn ": any\|as any\|any\[\| any;" apps/ops-api/src/ apps/ops-dashboard/ apps/sales-board/ packages/ --include="*.ts" --include="*.tsx" | grep -v node_modules | grep -v ".next/" | grep -v "__tests__" | grep -v "__mocks__" | grep -v ".d.ts" | wc -l
```

## Success Criteria

1. Zero `any` type annotations in application code (excluding test mocks and third-party stubs)
2. All shared package exports have explicit type annotations
3. TypeScript compilation succeeds without new errors
