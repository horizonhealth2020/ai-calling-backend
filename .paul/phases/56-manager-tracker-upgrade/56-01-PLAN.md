<objective>
## Goal
Add call quality metrics to the manager tracker — avg call length, longest call, and inline tier breakdown bar — by extending the existing `/call-counts` endpoint and upgrading ManagerTracker.tsx.

## Purpose
Managers currently see call QUANTITY (count per agent) but not call QUALITY. The ConvosoCallLog table already has callDurationSeconds for every call. This plan extracts tier breakdown (short/contacted/engaged/deep), avg call length, and longest call from the same local data, giving managers instant signal on who's having real conversations vs quick hangups.

## Output
- Extended `/call-counts` API returning tier breakdown + avg/longest call per agent
- ManagerTracker table with 3 new columns: Avg Call, Longest, Quality bar
- Inline stacked bar showing call quality distribution per agent
</objective>

  <action>
    Extend the existing `/call-counts` endpoint (lines 142-188) to compute per-agent tier breakdown, avg call length, and longest call from the local ConvosoCallLog data it already queries.

    **Current flow:** Fetches logs → filters by buffer → aggregates count by agent+leadSource → returns array.

    **New flow:** Same fetch + buffer filter, then ALSO compute per-agent aggregates:

    After the existing countMap aggregation loop, add a second aggregation loop that groups by agentId only:

    ```typescript
    // Per-agent call quality aggregation
    const agentQuality = new Map<string, { 
      tiers: { short: number; contacted: number; engaged: number; deep: number };
      totalDuration: number; 
      callCount: number; 
      longestCall: number; 
    }>();

    for (const log of logs) {
      const lsInfo = lsMap.get(log.leadSourceId!);
      const buffer = lsInfo?.callBufferSeconds ?? 0;
      if (buffer > 0 && log.callDurationSeconds !== null && log.callDurationSeconds < buffer) continue;
      
      const aid = log.agentId!;
      if (!agentQuality.has(aid)) agentQuality.set(aid, { 
        tiers: { short: 0, contacted: 0, engaged: 0, deep: 0 }, 
        totalDuration: 0, callCount: 0, longestCall: 0 
      });
      const q = agentQuality.get(aid)!;
      const dur = log.callDurationSeconds;
      if (dur === null) { /* live — skip tier */ }
      else if (dur < 30) q.tiers.short++;
      else if (dur < 120) q.tiers.contacted++;
      else if (dur < 300) q.tiers.engaged++;
      else q.tiers.deep++;
      if (dur !== null) {
        q.totalDuration += dur;
        q.callCount++;
        if (dur > q.longestCall) q.longestCall = dur;
      }
    }
    ```

    Add a new field to the response alongside the existing array:

    ```typescript
    const agentMetrics = Object.fromEntries(
      [...agentQuality.entries()].map(([id, q]) => [agentMap.get(id) ?? id, { <!-- audit-added: key by agentName, not agentId — ManagerTracker matches by name -->
        callsByTier: q.tiers,
        avgCallLength: q.callCount > 0 ? Math.round(q.totalDuration / q.callCount) : 0,
        longestCall: q.longestCall,
      }])
    );

    res.json({ counts: result, agentMetrics });
    ```

    **Breaking change handling:** The current response is a bare array. Wrapping it in `{ counts: [...], agentMetrics: {...} }` changes the shape. Update ManagerTracker to handle the new shape.

    Avoid: Changing the existing per-leadSource aggregation logic.
    Avoid: Calling the live Convoso API — use local ConvosoCallLog data only.
  </action>
