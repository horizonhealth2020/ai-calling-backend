<objective>
## Goal
Reverse the Phase 71 fronted formula decision: fronted is now a same-week deduction.
New formula: `Net = payout + adjustment + bonus - hold - fronted`.
Remove the carryover.ts fronted‚Üíhold logic (now dead code). Rewrite the 7-case
regression test suite to lock the new formula.

## Purpose
The current formula excludes fronted from net and carries it to the next period as hold
(via carryover.ts on period lock). This creates a multi-week deduction when the advance
was given same-week. The correct semantics: fronted reduces this week's net immediately,
the same way hold does.

## Output
- `computeNetAmount` deducts `fronted` from net
- `carryover.ts` no longer carries fronted‚Üíhold on period lock (logic removed)
- `payroll-net-formula.test.ts` has 7 cases locking the new formula
- All 184 existing tests still pass
- SUMMARY at `.paul/phases/78-payroll-polish/78-02-SUMMARY.md`
</objective>
