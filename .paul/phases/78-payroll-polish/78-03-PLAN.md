<objective>
## Goal
Five payroll UI polish items: (A) unapprove commission button gated to OPEN periods only,
(B) CS payroll card rep name bold/top/centered + Total green, (C) agent card entries sorted
by Sale.memberId ASC, (D) ACH print row green highlight matching on-screen, (F) week-level
payroll note box per agent-period — invisible on print when empty.

## Purpose
Payroll staff can currently unapprove commissions on locked periods (unintended). Agent card
entry order is arbitrary. ACH rows print white when they're green on-screen. CS payroll cards
lack visual hierarchy. There's no freeform note field on weekly payroll cards.

## Output
- Unapprove button hidden when period is LOCKED/FINALIZED or entry is paid-guarded
- CS payroll card: rep name bold, centered at top; Total label and amount green
- Agent card rows sorted by Sale.memberId ascending
- ACH rows print with green background (matching on-screen)
- Note box to the right of net amount — saves per agent-period — invisible on print when empty
- SUMMARY at `.paul/phases/78-payroll-polish/78-03-SUMMARY.md`
</objective>

<acceptance_criteria>

## AC-1: Unapprove button only shows for OPEN periods
```gherkin
Given a payroll entry has halvingReason and commissionApproved=true
When the period status is OPEN
Then the "Unapprove" button renders in WeekSection
And when the period status is LOCKED, FINALIZED, or the entry is PAID
Then the "Unapprove" button does NOT render (hidden, not just disabled)
```

## AC-2: Agent card entries sorted by Sale.memberId ascending
```gherkin
Given an agent has multiple sale entries in a pay period week card
When the week card renders
Then the entries are ordered by Sale.memberId ascending (string sort, e.g. "100" before "200")
And entries with null memberId sort last (or first — consistent)
```

## AC-3: ACH print row green to match on-screen
```gherkin
Given a payroll entry has paymentType="ACH"
When the user prints the payroll card (Ctrl+P or browser print)
Then the row shows a green background (matching the on-screen green row highlight)
And non-ACH rows print without a colored background
```

## AC-4: Week-level note box saves and renders correctly
```gherkin
Given a payroll staff member is viewing an agent's week card
When they type text into the note textarea (positioned right of net amount)
And click save (or blur, whichever is implemented)
Then PATCH /api/payroll/adjustments/:id saves the text to AgentPeriodAdjustment.notes
And on next render the note textarea shows the saved text
And when printing: if the note is empty, the textarea is invisible on print
And when printing: if the note has text, the note is visible on print
```

## AC-5: CS payroll card cosmetics
```gherkin
Given the CS payroll section in PayrollService.tsx
When a service agent's payroll card renders
Then the rep name is bold, positioned at the top, and horizontally centered
And the "Total" label and total pay amount are rendered in green (colors.success or semanticColors equivalent)
And all other card content (base pay, bonus, deductions) is unchanged
```
