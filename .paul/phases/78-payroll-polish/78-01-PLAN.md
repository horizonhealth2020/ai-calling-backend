<objective>
## Goal
Fix the sale edit bug cluster: missing addon premium inputs in the edit form, [object Object]
diff display in the CHANGES section, and the string error managers receive when saving
edits that include addon changes.

## Purpose
Managers and payroll staff are blocked from correctly editing addon premiums on existing
sales — the edit form has no premium fields for checked addons, the CHANGES diff renders
objects as "[object Object]", and saving an edit with addon changes returns a string error.
Commission recalculation after approval depends on correct addon premium data.

## Output
- Addon premium inputs in edit form for each checked addon (matching ManagerEntry.tsx pattern)
- CHANGES section displays object values as JSON (not [object Object])
- Manager can save an edit with addon changes without error
- Commission recalculates with correct addon premiums after PAYROLL approves the edit
- SUMMARY at `.paul/phases/78-payroll-polish/78-01-SUMMARY.md`
</objective>
