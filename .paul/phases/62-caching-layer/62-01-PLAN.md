<objective>
## Goal
Create an in-memory cache service and wire it into the 5 heaviest aggregation endpoints to eliminate lag when switching dashboard tabs. Auto-invalidate on data mutations via existing Socket.IO emit functions and direct invalidation calls in mutation handlers.

## Purpose
Dashboard tab switches trigger full DB aggregation queries every time. With 18 agents and growing data, these queries cause noticeable lag. Caching serves repeated requests from memory and invalidates on mutations.

## Output
- `apps/ops-api/src/services/cache.ts` — TTL-based in-memory cache with invalidation API
- 5 endpoints wrapped with cache: command-center, tracker/summary, owner/summary, owner/trends, reporting/periods
- Socket.IO emit functions + uncovered mutation handlers trigger cache invalidation
</objective>

  <name>Task 1: Create in-memory cache service</name>
  <files>apps/ops-api/src/services/cache.ts</files>
  <action>
    Create `apps/ops-api/src/services/cache.ts`:

    Simple Map-based cache with TTL:

    ```ts
    const store = new Map<string, { data: unknown; expiresAt: number }>();
    const inflight = new Map<string, Promise<unknown>>(); // audit-added: stampede protection
    const DEFAULT_TTL_MS = 30_000; // 30 seconds

    export function cacheGet<T>(key: string): T | undefined
    export function cacheSet(key: string, data: unknown, ttlMs?: number): void
    export function invalidate(pattern: string): void  // removes keys starting with pattern
    export function invalidateAll(): void
    export function cacheWrap<T>(key: string, fn: () => Promise<T>, ttlMs?: number): Promise<T>
    export function cacheStats(): { entries: number; hits: number; misses: number }
    ```

    **cacheWrap** is the main API — wraps an async function:
    - If cache hit (not expired): return cached value, increment hits counter
    - If cache miss but in-flight request exists for same key: return the in-flight Promise (stampede protection) <!-- audit-added -->
    - If cache miss and no in-flight: call fn(), store Promise in inflight map, on resolve → cache result + remove from inflight, on reject → remove from inflight WITHOUT caching the error <!-- audit-added -->
    - Log cache hit/miss to console at debug level: `[cache] HIT key` / `[cache] MISS key` <!-- audit-added -->

    **invalidate(pattern)** removes all keys that start with `pattern`:
    - `invalidate("sales:")` removes `sales:/tracker/summary?range=week`, `sales:/command-center?range=today`, etc.
    - Also clears matching inflight entries <!-- audit-added -->

    **Key format convention:** `{route-module}:{path}?{query-string}`
    - Example: `sales:/command-center?range=week`
    - Example: `sales:/tracker/summary?range=today`
    - Example: `admin:/owner/trends?days=30`

    **Cleanup:** Set an interval (every 60s) to sweep expired entries from the map to prevent memory growth. Use `setInterval` with `unref()` so it doesn't keep the process alive.

    **Stats tracking:** Maintain hit/miss counters for cacheStats(). Useful for debugging performance.
  </action>
