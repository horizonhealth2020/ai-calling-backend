<objective>
## Goal
Make the payroll dashboard's three highest-traffic surfaces — agent selection (`AgentSidebar` inside `PayrollPeriods`), per-agent pay card (`AgentCard`), and weekly breakdown with adjustments (`WeekSection`) — usable on a 375px phone. Sidebar moves into a `MobileDrawer`, AgentCard KPIs stack 1-column with full-width cards, WeekSection tables become responsive cards, all dollar-amount inputs get touch-target sizing. Net-formula display values must remain accurate to the cent. Print-view HTML strings (lines 902 + 993 in PayrollPeriods.tsx) are explicitly not touched.

## Purpose
Payroll is operated by 1-2 staff who currently work from a desktop. The Friday payroll close happens once a week; doing a quick correction or status check from a phone is the realistic mobile use case. Two specific things matter most: (a) the manager / payroll lead can scan an agent's commission + chargebacks + adjustments without horizontal scroll, and (b) editing an adjustment dollar amount on a phone doesn't fat-finger the number because the input is too small. ManagerAudits-equivalent in payroll (PayrollChargebacks, PayrollProducts, PayrollExports, PayrollService) deferred — financial admin work that doesn't need a mobile retrofit yet.

## Output
- AgentSidebar accessible via a mobile-only "Select Agent" button at the top of the content area; on mobile the sidebar renders inside a `MobileDrawer side="left"` instead of taking 220px of viewport
- LAYOUT container collapses to `flex-direction: column` on mobile (was `display: flex` row); sidebar element's standalone `display: flex` block wrapper is hidden on mobile via `.hide-mobile`, replaced by the drawer trigger
- AgentCard: KPI strip stacks 1-column on mobile; per-agent header info wraps cleanly; touch-target on every action button
- WeekSection tables: `.responsive-table` + data-labels for the agent-week breakdown table at line 1062; inline adjustment-amount inputs get `.touch-target` (44px min height — protects against fat-finger entry of dollar amounts)
- Period selector header strip: stacks vertically on mobile; touch-target on lock/unlock/finalize buttons
- Sidebar drawer trigger uses the existing project-standard scroll-affordance pattern internally where a wide list overflows
- Net-formula computed display values verified to render unchanged (regression: same `formatDollar(...)` calls on the same data)
</objective>
