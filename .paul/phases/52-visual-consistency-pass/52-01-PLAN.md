<objective>
## Goal
Extend the design token system with semantic color aliases for the 30 unique hardcoded hex values, fix responsive grids on KPI cards, and add touch support to sidebar navigation — laying the foundation for Plan 52-02's bulk color/typography migration.

## Purpose
The token system in tokens.ts has comprehensive base tokens (colors, typography, spacing) but dashboards use ~30 hardcoded hex values that map to these tokens. This plan adds semantic aliases (statusRan, statusDead, etc.) so Plan 52-02 can do a clean find-and-replace migration. Responsive grids and touch nav are quick wins that complete the ROADMAP scope.

## Output
- Extended tokens.ts with semantic color aliases covering all 30 hardcoded hex values
- Responsive KPI grids on CS and owner dashboards
- Touch-friendly sidebar navigation in layout.tsx and PageShell
</objective>

  <files>apps/ops-dashboard/app/(dashboard)/layout.tsx, packages/ui/src/index.tsx</files>
  <action>
    **layout.tsx (dashboard tab navigation):**
    The nav currently uses `onMouseEnter`/`onMouseLeave` to expand/collapse:
    - Line 93: `onMouseEnter={() => setHovered(true)}`
    - Line 94: `onMouseLeave={() => setHovered(false)}`

    Add touch support:
    - Add `onClick` handler that toggles `hovered` state (touch devices trigger click, not hover)
    - The existing mouse handlers continue working for desktop
    - This makes the nav toggleable on touch via tap

    **PageShell (packages/ui/src/index.tsx):**
    Check for mouse-only handlers on nav items and sidebar. If found:
    - Add equivalent touch/click handlers
    - Ensure sidebar can be toggled on touch devices

    **Pattern:** Use `onTouchEnd` (not `onClick`) to scope toggle to touch devices only: <!-- audit-added: onClick fires on desktop clicks too, causing stuck-collapsed nav -->
    ```
    onTouchEnd={(e) => { e.preventDefault(); setHovered(h => !h); }}  // toggle on tap only
    onMouseEnter={() => setHovered(true)} // expand on hover (desktop)
    onMouseLeave={() => setHovered(false)} // collapse on leave (desktop)
    ```

    The `e.preventDefault()` stops the touch event from also firing a synthetic mouse event.

    Avoid: Using `onClick` for touch toggle — it fires on desktop mouse clicks too, causing nav to get stuck collapsed when desktop user accidentally clicks.
    Avoid: Adding complex gesture libraries — simple onTouchEnd is sufficient.
  </action>
  <verify>
    - Desktop: nav expands on hover, collapses on mouse leave (unchanged)
    - Touch/mobile: nav toggles on tap (onTouchEnd)
    - Desktop: clicking nav does NOT toggle (only hover works) <!-- audit-added -->
    - Chrome DevTools mobile emulation: tap to expand, tap again to collapse
  </verify>
  <done>AC-3 satisfied: Navigation supports both mouse hover and touch tap</done>
