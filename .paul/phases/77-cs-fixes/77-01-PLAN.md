<objective>
## Goal
Close three CS gaps: (A) tracking list hides new submissions when prior resolved rows exist for same member, (B) no submission-time dedupe lets accidental re-pastes slip in, (C) 6 CS reps with active logins have empty MyQueue due to name-string filter failing against unlinked User/CsRepRoster entities.

## Purpose
Unblocks CS rep daily workflow (MyQueue + stale alerts) and prevents accidental-duplicate submissions without blocking legitimate same-week re-submissions. Structural fix for rep identity (FK) replaces fragile name-string matching.

## Output
- Each submission appears as its own tracking row regardless of prior member history
- Server-side dedupe on composite keys; preview UI surfaces duplicates as warnings
- `User.csRepRosterId` FK linking CUSTOMER_SERVICE users to `CsRepRoster` entries
- Owner UI dropdown to create/maintain the link
- Auto-sync on new user creation
- `/api/stale-summary` resolves filter via FK, not session `User.name`
- SUMMARY at `.paul/phases/77-cs-fixes/77-01-SUMMARY.md`
</objective>
