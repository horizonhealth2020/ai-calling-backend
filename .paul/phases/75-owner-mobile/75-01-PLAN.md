<objective>
## Goal
Make the owner dashboard's four tab-content surfaces — Command Center (OwnerOverview), Trends (OwnerTrends), KPIs (OwnerKPIs), and Scoring (OwnerScoring) — usable on a 375px phone. Leaderboard and KPI tables become responsive cards, stat card grids stack 1-column, Recharts charts get mobile-safe margins, header bars stack vertically, and activity feed items truncate cleanly. OwnerConfig and OwnerUsers are admin-only settings forms — deferred (not high-traffic mobile surfaces).

## Purpose
The owner occasionally checks office performance from a phone — a quick glance at today's premium, the leaderboard ranking, or a scoring trend. The primary use case is consumption (scanning KPIs, reading the feed) not data entry. Two things matter: (a) stat cards + leaderboard readable without horizontal scroll, and (b) Recharts charts render without clipping axis labels or losing the legend.

## Output
- OwnerOverview: header bar stacks vertically, stat cards grid stacks 1-col, leaderboard table becomes responsive cards, activity feed items get touch-target padding, hero section stacks column
- OwnerTrends: chart sections stack vertically (already do), chart margins reduced for narrow viewport, agent selector drops below header on mobile, date range filter stacks
- OwnerKPIs: header stacks vertically, agent KPI table becomes responsive cards
- OwnerScoring: header stacks vertically, per-agent score table + weekly trends table become responsive cards, stat cards already have grid-mobile-1
</objective>
