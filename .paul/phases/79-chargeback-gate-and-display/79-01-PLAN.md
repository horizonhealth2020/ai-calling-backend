<objective>
## Goal

Fix the chargeback approval gate so CS-submitted chargebacks only mutate the agent paycard **on payroll approval**, not on submission. In parallel, fix the cross-period paycard display bugs that make the approved negative row invisible — `formatDollar` strips the sign, agent net/subtotal don't include `adjustmentAmount`, sidebar net still uses Phase 71 formula residue, and `CLAWBACK_CROSS_PERIOD` row tint is orange instead of red.

## Purpose

Payroll correctness: agents are currently being docked at CS submission time for chargebacks that have not yet been reviewed by payroll. That inverts the intended approval gate (v1.2). Once a chargeback IS approved (or entered payroll-directly), the -$X row must be visibly deducted from the on-screen agent total so payroll staff can reconcile without doing arithmetic by hand. Both bugs together explain the Victoria Checkal symptom in `image.png` (repo root): `$76.04` in red without a minus sign, row with peach tint, net `$679.11` (sum of the other 7 rows, Victoria invisible to the math).

## Output

- `POST /chargebacks` gates Clawback + applyChargebackToEntry on `source !== "CS"`; CS path only creates the PayrollAlert. Approval via `alerts.ts:approveAlert` becomes the sole paycard-mutation trigger for CS-originated chargebacks.
- `formatDollarSigned()` utility added to `packages/utils` (leading minus format). `formatDollar` stays untouched to protect ~40 unrelated call sites.
- Agent-card subtotal, liveNet, sidebar net, and print net all include per-entry `adjustmentAmount` and use Phase 78 formula (`- fronted`, not `+ fronted`). Screen aligns to the already-correct print view.
- `CLAWBACK_CROSS_PERIOD` row tint → red (`semanticColors.statusDead` @ 0.08); `ZEROED_OUT_IN_PERIOD` stays yellow.
- `.paul/phases/79-chargeback-gate-and-display/79-01-SUMMARY.md` produced at UNIFY.
