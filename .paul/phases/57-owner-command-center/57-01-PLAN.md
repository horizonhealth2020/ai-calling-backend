<objective>
## Goal
Create two new API endpoints for the Owner Command Center: a single aggregated `/api/command-center` endpoint that returns all metrics in one call, and a `/api/activity-feed` endpoint that returns recent CS and manager actions from the audit log.

## Purpose
The command center needs data from 5+ existing endpoints (tracker, chargebacks, payroll, call counts, owner summary). Rather than making 5 parallel fetches from the frontend, a single `/api/command-center` endpoint aggregates everything server-side. The activity feed needs a new endpoint since AppAuditLog has no GET route.

## Output
- `GET /api/command-center` — single response with hero metrics, stat cards, leaderboard data
- `GET /api/activity-feed` — recent audit log events filtered to CS/manager actions
</objective>


```gherkin
Given an authenticated OWNER_VIEW or SUPER_ADMIN user
When they call GET /api/activity-feed?limit=20

  - Each event has: id, action, entityType, entityId, metadata, createdAt, actorName
  - Filtered to: Sale CREATE/UPDATE, ChargebackSubmission CREATE, PendingTerm CREATE, ChargebackSubmission UPDATE (resolve/unresolve)
```
