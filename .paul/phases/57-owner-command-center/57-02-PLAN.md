<acceptance_criteria>

## AC-1: Hero section displays premium and sales with delta
```gherkin
Given the command center loads with data from /api/command-center
When the hero section renders
Then it shows Total Premium as a large animated number
And Total Sales count alongside it
And a period selector (today/week/last week/30 days)
And delta vs prior period as percentage with up/down arrow and color (green=up, red=down)
```

## AC-2: Four stat cards with actionable context
```gherkin
Given command center data loaded
When stat cards render
Then "This Week Premium" shows dollar amount with vs last week delta
And "Commission Owed Friday" shows arrears period total
And "Chargebacks" shows count + dollar, red border pulse if trending up vs last week
And "Lead ROI" shows avg cost per sale with vs last week delta
And cards with no Convoso data show "—" for lead metrics
```

## AC-3: Condensed agent leaderboard
```gherkin
Given leaderboard data from /api/command-center
When the leaderboard table renders
Then it shows: Rank, Agent, Calls, Avg (m:ss), Sales, Premium, Cost/Sale, Quality dot
And top 3 agents have rank icons (trophy/medal/award)
And quality dot is green (>60% engaged+deep), amber (30-60%), red (<30%)
And the table is sorted by premium descending
```

## AC-4: Responsive phone layout
```gherkin
Given the command center viewed on a phone (<768px viewport)
When the layout adapts
Then hero section stacks vertically (premium on top, sales below)
And stat cards arrange in 2x2 grid
And leaderboard is horizontally scrollable or condensed
And all text remains readable without pinch-to-zoom
```

## AC-5: Loading and empty states
```gherkin
Given the command center is loading data
When the API call is in flight
Then skeleton placeholders show for hero, stat cards, and leaderboard
Given no data exists (new account, no sales)
When the page renders
Then appropriate empty states display with helpful messaging
```

    - Use responsive grid: `repeat(auto-fit, minmax(200px, 1fr))`
    - Each card: glassmorphism surface, subtle border, left color accent strip (3px)
    - **This Week Premium:** accent=teal, shows dollar + delta badge
    - **Commission Friday:** accent=blue, shows dollar amount (from commissionOwedFriday)
    - **Chargebacks:** accent=red when trending up (chargebackCount > priorChargebackCount), else default. Shows "count ($X)" format. If trending up: `animation: pulse 2s infinite` on border

    **Condensed Leaderboard:**
    - Card with table inside
    - Headers: Rank, Agent, Calls, Avg, Sales, Premium, Cost/Sale, Quality
    - Compact padding (8px 12px cells)
    - Top 3: Trophy/Medal/Award icons with subtle background tint
    - Quality dot component:
      ```
      function QualityDot({ tiers }) {
        const total = tiers.short + tiers.contacted + tiers.engaged + tiers.deep;
        if (total === 0) return <span>—</span>;
        const goodRatio = (tiers.engaged + tiers.deep) / total;
        const color = goodRatio > 0.6 ? semanticColors.accentGreenMid
          : goodRatio > 0.3 ? semanticColors.warningAmber
          : semanticColors.dangerLight;
        return <span title={`${Math.round(goodRatio*100)}% engaged/deep`} style={{ width:8, height:8, borderRadius:'50%', background:color, display:'inline-block' }} />;
      }
      ```
    - Duration formatter: reuse `fmtDuration` pattern from ManagerTracker

    **Responsive:**
    - Hero: flexDirection column on phone (`@media (max-width: 768px)` via inline check or container width detection)
    - Since we use inline styles (no CSS media queries), use a `useWindowWidth()` hook or check `typeof window !== 'undefined' && window.innerWidth < 768`
    - Stat cards: 2x2 on phone (auto-fit handles this with minmax(200px))
    - Leaderboard: add `overflowX: 'auto'` wrapper

    **Skeleton:**
    - Reuse existing `SkeletonCard` from @ops/ui for loading state
    - Show 1 tall skeleton for hero + 4 skeleton cards for stat row + 1 tall skeleton for leaderboard

    Avoid: Keeping any of the old OwnerOverview code — this is a full rewrite.
