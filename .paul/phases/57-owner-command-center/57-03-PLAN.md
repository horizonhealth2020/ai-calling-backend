<objective>
## Goal
Add a live activity feed to the Owner Command Center that shows recent CS and manager actions, loading initial events from /api/activity-feed and streaming new ones via Socket.IO.

## Purpose
The command center shows metrics but not WHAT is happening. The activity feed answers "what just happened in my office?" — sales entered, chargebacks filed, terms submitted, resolutions made. Real-time via Socket.IO so the owner sees events as they occur.

## Output
- ActivityFeed component rendered below the leaderboard in OwnerOverview
- Initial load from /api/activity-feed (last 20 events)
- Real-time: new events prepended via Socket.IO listeners
- Filtered to CS + manager actions only
</objective>

    ```typescript
    function formatFeedEvent(event: FeedEvent): { icon: ReactNode; description: string } {
      const d = event.details as Record<string, string | number | undefined>;
      if (event.entityType === "Sale" && event.action === "CREATE") {
        return { icon: <DollarSign size={14} />, description: `entered a sale — ${d.productName ?? "Sale"} ${d.premium ? fmtDollar(Number(d.premium)) : ""} for ${d.memberName ?? ""}` };
      }
      if (event.entityType === "Sale" && event.action === "UPDATE") {
        return { icon: <Edit3 size={14} />, description: `updated sale — ${d.memberName ?? ""}` };
      }
      if (event.entityType === "ChargebackSubmission" && event.action === "CREATE") {
        return { icon: <AlertTriangle size={14} />, description: `filed chargeback — ${d.payeeName ?? ""} ${d.totalAmount ? fmtDollar(Number(d.totalAmount)) : ""}` };
      }
      if (event.entityType === "ChargebackSubmission" && event.action === "UPDATE") {
        return { icon: <CheckCircle size={14} />, description: `resolved chargeback — ${d.payeeName ?? ""}` };
      }
      if (event.entityType === "PendingTerm" && event.action === "CREATE") {
        return { icon: <Clock size={14} />, description: `submitted pending term — ${d.memberName ?? ""}` };
      }
      return { icon: <Activity size={14} />, description: `${event.action.toLowerCase()} ${event.entityType}` };
    }
    ```

    **Relative timestamp:** Format createdAt as "2m ago", "1h ago", "yesterday" etc.
    ```typescript
    function timeAgo(dateStr: string): string {
      const diff = Date.now() - new Date(dateStr).getTime();
      const mins = Math.floor(diff / 60000);
      if (mins < 1) return "just now";
      if (mins < 60) return `${mins}m ago`;
      const hrs = Math.floor(mins / 60);
      if (hrs < 24) return `${hrs}h ago`;
      const days = Math.floor(hrs / 24);
      return `${days}d ago`;
    }
    ```

    **Socket.IO listeners — refetch on activity:** <!-- audit-added: socket payloads lack actorName, refetch ensures correct data -->
    Instead of synthesizing events from socket payloads (which lack actorName and full details),
    refetch the activity feed when relevant socket events fire:
    - `sale:changed` → refetch /api/activity-feed
    - `cs:changed` → refetch /api/activity-feed

    ```typescript
    const refetchFeed = useCallback(() => {
      authFetch(`${API}/api/activity-feed?limit=20`)
        .then(r => r.ok ? r.json() : { events: [] })
        .then(data => setFeedEvents(data.events ?? []));
    }, [API]);

    useEffect(() => {
      if (!socket) return;
      socket.on("sale:changed", refetchFeed);
      socket.on("cs:changed", refetchFeed);
      return () => { socket.off("sale:changed", refetchFeed); socket.off("cs:changed", refetchFeed); };
    }, [socket, refetchFeed]);
    ```

    This ensures actorName and entity details are always correct (resolved from DB, not guessed from socket payload).
