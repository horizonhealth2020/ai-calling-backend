<objective>
## Goal
Replace the static OwnerKPIs tab with a real-time Trends tab featuring Recharts-powered analytical charts — agent KPI trends, revenue trends, lead source effectiveness, and call quality distribution over time.

## Purpose
Owners need to see trends, not just snapshots. The current KPIs tab shows a flat table of chargeback/pending term counts with no historical context. Real charts with axes, tooltips, and legends let owners spot patterns, compare agents, and make data-driven decisions about lead sources and agent coaching.

## Output
- New `/api/owner/trends` API endpoint returning time-series aggregations
- OwnerTrends.tsx component with 4 Recharts chart sections
- KPIs tab renamed to "Trends" in owner page navigation
- Recharts added as ops-dashboard dependency
</objective>

## AC-1: Trends API returns time-series data
```gherkin
Given an authenticated OWNER_VIEW or SUPER_ADMIN user
When they call GET /api/owner/trends?range=month (or from/to params)
Then the response contains:
  - revenueTrend: array of { date, premiumTotal, commissionTotal, chargebackTotal } per week
  - agentKpiTrend: array of { date, agentId, agentName, totalCalls, avgCallLength, closeRate } per snapshot date
  - leadSourceEffectiveness: array of { sourceId, sourceName, salesCount, premiumTotal, callCount, costPerSale, conversionRate }
  - callQualityTrend: array of { date, short, contacted, engaged, deep } per snapshot date
And all numeric values are serialized as numbers (Decimal fields converted via Number())
```

## AC-1b: Trends API handles missing/sparse data gracefully <!-- audit-added -->
```gherkin
Given an authenticated user calls GET /api/owner/trends?range=month
When the date range has no AgentCallKpi snapshots or no Sales
Then each array returns empty [] (not null, not undefined)
And no 500 error is thrown
And Decimal fields with null values default to 0
```

## AC-1c: Trends API rejects unauthenticated and unauthorized requests <!-- audit-added -->
```gherkin
Given an unauthenticated request or a user with role other than OWNER_VIEW/SUPER_ADMIN
When they call GET /api/owner/trends
Then the response is 401 or 403 respectively
```

  <name>Task 1: Trends API endpoint and aggregation service</name>
  <files>apps/ops-api/src/services/trendAggregator.ts, apps/ops-api/src/routes/admin.ts</files>
  <action>
    Create new service `trendAggregator.ts` with function `getOwnerTrends(dateRange)`:

    **IMPORTANT: The `dateRange` helper returns `{ gte: Date, lt: Date }` (not `from`/`to`). Use these field names in all Prisma where clauses.** <!-- audit-added -->

    1. **revenueTrend** — Query Sales (status = RAN) grouped by week (using saleDate truncated to Monday — use saleDate not createdAt, since saleDate is the business date). For each week: sum premium (convert Decimal to Number), sum commission from PayrollEntry (join via Sale.payrollEntries), sum chargeback amounts from ChargebackSubmission (join via matchedSaleId). Return array sorted by week ascending. Handle empty weeks: include them with zero values so chart shows continuous timeline. <!-- audit-amended: saleDate not createdAt, Decimal conversion, empty week handling -->

    2. **agentKpiTrend** — Query AgentCallKpi snapshots within date range. Group by snapshotDate (truncated to day) and agentId. For each: sum totalCalls, avg of avgCallLength (convert Decimal to Number), compute closeRate as (RAN sales count for that agent in date range / totalCalls). Join agent names from Agent model. Return array sorted by date. **Guard against division by zero: if totalCalls is 0, closeRate = 0.** <!-- audit-amended: division-by-zero guard -->

    3. **leadSourceEffectiveness** — Query Sales (status = RAN) grouped by leadSourceId within date range. Join LeadSource names. For each: count sales, sum premium (Decimal → Number). Query ConvosoCallLog for call counts per leadSourceId (note: leadSourceId is nullable on ConvosoCallLog, skip nulls). Query AgentCallKpi for totalLeadCost per leadSourceId (Decimal → Number, null → 0). Compute conversionRate (sales/calls, guard div-by-zero) and costPerSale (leadCost/sales, guard div-by-zero). Exclude sources with 0 sales. <!-- audit-amended: nullable leadSourceId, Decimal conversion, div-by-zero guards -->

    4. **callQualityTrend** — Query AgentCallKpi snapshots within date range grouped by snapshotDate. Parse callsByTier JSON (type: `{ short: number, contacted: number, engaged: number, deep: number }`), aggregate tier counts across all agents per date. **Validate JSON shape before accessing keys — if a snapshot has malformed callsByTier, skip it rather than throwing.** Return array of { date, short, contacted, engaged, deep } sorted ascending. <!-- audit-amended: JSON validation -->

    All queries use the `dateRange()` helper which returns `{ gte, lt }` — pass directly as Prisma where clause on date fields.
    Use existing `dateRangeQuerySchema` from `./helpers`.
    Use Promise.all for parallel queries where possible.
    **Wrap the entire function in try/catch — if any sub-query fails, return the successful results with empty arrays for failed sections, and log the error via logError.** <!-- audit-added: partial failure resilience -->

    Add route in admin.ts:
    ```
    GET /api/owner/trends?range=[preset]|from=&to=
    Roles: OWNER_VIEW, SUPER_ADMIN (via requireAuth + requireRole)
    ```
    Validate with dateRangeQuerySchema, call getOwnerTrends, return JSON.
    **If dateRange returns undefined (no valid range provided), return 400 with zodErr-style error.** <!-- audit-added: missing range guard -->
