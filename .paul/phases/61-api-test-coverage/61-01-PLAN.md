    Create `apps/ops-api/src/services/__tests__/chargeback-flow.test.ts`:

    Uses the same pattern as existing tests — import functions directly, mock @ops/db via jest.mock.

    **Test structure:**

    ```
    describe('applyChargebackToEntry', () => {
      describe('in-period (OPEN)', () => {
        - zeros payoutAmount and netAmount on original entry
        - sets status to ZEROED_OUT_IN_PERIOD
        - returns { mode: "in_period", entryId }
      })

      describe('cross-period (LOCKED)', () => {
        - does not modify original entry
        - creates new entry with payoutAmount: 0, adjustmentAmount: -amount
        - new entry has status CLAWBACK_CROSS_PERIOD
        - new entry is in the oldest OPEN period
        - returns { mode: "cross_period", entryId }
      })

      describe('entry filtering', () => {
        - skips entries with CLAWBACK_APPLIED status
        - skips entries with ZEROED_OUT_IN_PERIOD status
        - skips entries with CLAWBACK_CROSS_PERIOD status
        - selects oldest non-clawback entry by createdAt ascending
      })

      describe('error cases', () => {
        - throws when no eligible live entry exists (all already clawed back)
        - throws when no OPEN period exists for cross-period
        - error messages include sale ID for debugging
      })
    })
    ```

    **Mock setup — use TransactionClient mock pattern:**
