Fix chargeback net deduction display, add missing print view status colors, and support a new tab-separated chargeback batch parser format.

## Purpose
Cross-period chargebacks create negative adjustment entries but their adjustmentAmount is not included in the agent net calculation — so the deduction doesn't show in the agent's displayed net payout. Print view is also missing ACH green and CLAWBACK_APPLIED red row colors that exist on screen. Additionally, CS staff need to paste a simpler chargeback list format (policy ID + member name) alongside the existing financial report format.

## Output
- Agent net payout correctly reflects cross-period chargeback deductions (on-screen and print)
- Print view shows all 4 status row colors matching on-screen (ACH green, CLAWBACK_APPLIED red, plus existing orange/yellow)
- New simple chargeback list format auto-detected and parsed alongside existing format
</objective>

    **1. Add format detection function** (before parseChargebackText):
    ```typescript
    function isSimpleChargebackFormat(raw: string): boolean {
      const lines = raw.split("\n").filter(l => l.trim().length > 0);
      if (lines.length === 0) return false;
      // Simple format: starts with a numeric ID, followed by name parts only
      // Financial format: has dates (D-MMM-YY), dollar amounts ($), percentages
      const sample = lines.slice(0, Math.min(3, lines.length));
      return sample.every(line => {
        const fields = line.trim().split(/\t+|\s{2,}/);
        // Must start with numeric ID (6+ digits), rest are name parts (no $, no dates, no %)
        return fields.length >= 2
          && /^\d{6,}$/.test(fields[0])
          && !fields.some(f => /^\d{1,2}-[A-Za-z]{3}-\d{2}$/.test(f) || /\$/.test(f) || /%/.test(f));
      });
    }
    ```

    **2. Add simple format parser** (after isSimpleChargebackFormat):
    ```typescript
    function parseSimpleChargebackList(raw: string): ConsolidatedRecord[] {
      const lines = raw.split("\n").filter(l => l.trim().length > 0);
      return lines.map(line => {
        const fields = line.trim().split(/\t+|\s{2,}/);
        const memberId = fields[0] || "";
        // Remaining fields are name parts — join them as the full name
        const memberName = fields.slice(1).join(" ").trim();
        return {
          postedDate: null,
          type: "Chargeback",
          payeeId: null,
          payeeName: memberName, // <!-- audit-added --> Store parsed name for preview display and name-based matching
          payoutPercent: null,
          chargebackAmount: 0, // Server defaults to full commission when 0 (see audit-added API change)
          totalAmount: 0,
          transactionDescription: null,
          product: "",
          memberCompany: memberName, // <!-- audit-added --> Fallback key for consolidateByMember grouping
          memberId,
          memberAgentCompany: null,
          memberAgentId: null,
          assignedTo: "",
        };
      });
    }
    ```

    Note: chargebackAmount is 0 because the simple format has no dollar amounts. The server-side chargebacks route must default to the matched sale's full commission when chargebackAmount is 0 (see audit-added API task below).

    **3. Update handleTextChange** (line ~452-461) to auto-detect format:
