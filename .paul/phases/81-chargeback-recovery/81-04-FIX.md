<objective>
## Goal
Fix unresolve round-trip gap: after CS resolves "recovered" (auto-clearing an orphan SUBMISSION alert via M-3) and then unresolves, the SUBMISSION alert stays CLEARED so payroll has no visible alert to act on.

## Purpose
Discovered during 81-03 re-UAT:
- Submit chargeback → SUBMISSION alert PENDING
- CS mark recovered (before payroll approves) → AC-2 / M-3 auto-clears the alert; submission is "recovered"
- CS unresolve → PATCH /unresolve only nulls submission resolution fields; the CLEARED alert is not touched
- Payroll has no alert to approve → chargeback is effectively forgotten

## Output
- `PATCH /chargebacks/:id/unresolve` wrapped in transaction: nulls resolution fields AND un-clears the most-recent CLEARED SUBMISSION alert for this submission (resets to PENDING, clears clearedAt/clearedBy).
- Adds `logAudit` entries on unresolve (both the resolution clear and the alert un-clear if applicable).

Source: 2026-04-17 UAT on 81-03 re-test.
Priority: 1 Major (workflow round-trip correctness).
</objective>

    ```
    router.patch("/chargebacks/:id/unresolve", requireAuth, requireRole("CUSTOMER_SERVICE", "SUPER_ADMIN", "OWNER_VIEW"), asyncHandler(async (req, res) => {
      const pp = idParamSchema.safeParse(req.params);
      if (!pp.success) return res.status(400).json(zodErr(pp.error));

      const result = await prisma.$transaction(async (tx) => {
        const record = await tx.chargebackSubmission.update({
          where: { id: pp.data.id },
          data: {
            resolvedAt: null,
            resolvedBy: null,
            resolutionNote: null,
            resolutionType: null,
          },
        });

        // Phase 81-04: Round-trip correctness for recovery workflow.
        // If a SUBMISSION alert was cleared (either auto via M-3 orphan-recovery-cleanup OR manually by payroll),
        // un-clear it on unresolve so payroll has a visible alert to act on again.
        const clearedSubmission = await tx.payrollAlert.findFirst({
          where: {
            chargebackSubmissionId: pp.data.id,
            type: "SUBMISSION",
            status: "CLEARED",
          },
          orderBy: { clearedAt: "desc" },
        });
        let reinstatedAlertId: string | null = null;
        if (clearedSubmission) {
          await tx.payrollAlert.update({
            where: { id: clearedSubmission.id },
            data: { status: "PENDING", clearedAt: null, clearedBy: null },
          });
          reinstatedAlertId = clearedSubmission.id;
        }

        return { record, reinstatedAlertId };
      });

      // Post-commit logAudit.
      logAudit(req.user!.id, "UPDATE", "ChargebackSubmission", pp.data.id, { action: "unresolve" });
      if (result.reinstatedAlertId) {
        logAudit(req.user!.id, "submission_alert_reinstated_on_unresolve", "PayrollAlert", result.reinstatedAlertId, {
          chargebackSubmissionId: pp.data.id,
        });
      }

      return res.json(result.record);
    }));
    ```
