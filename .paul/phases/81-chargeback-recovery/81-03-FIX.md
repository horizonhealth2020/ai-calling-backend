<objective>
## Goal
Fix UAT-005: CS resolve "recovered" silently no-ops when `ChargebackSubmission.matchedSaleId` is null (orphaned after sale delete + re-entry), even though a matching Clawback DOES exist for the same member.

## Purpose
Root cause discovered via live DB audit trace on 2026-04-17 (Donna Zarembski case):
- Original sale deleted → `ChargebackSubmission.matched_sale_id = null`, `match_status = 'MATCHED'` (stale)
- User re-entered sale + payroll-staff POSTed `/api/clawbacks` → NEW Clawback created with `matchedBy='member_id'`, `matchedValue=memberId`, linked to the new sale
- CS marks "recovered" → Phase 81 handler's Clawback-finder ONLY activates member_id/member_name branches when `record.matchedSaleId` is truthy. Since it's null, only the `chargeback_alert` branch runs, which misses this Clawback → `chargeback_resolved_recovered_no_clawback` → no RECOVERY alert → agent stays docked despite money recovered.

## Output
- Widen Clawback-finder in `PATCH /chargebacks/:id/resolve` to add a fallback branch when `matchedSaleId` is null: match by `memberId` or `memberCompany` alone (unscoped by saleId).
- Order results by `createdAt desc` to prefer the most recent Clawback.

Source: `.paul/phases/81-chargeback-recovery/81-01-UAT.md` (UAT-005)
Priority: 1 Major (data-inconsistency path — recovery flow silently fails for re-entered sales)
</objective>

    ```
    const clawbackWhereOrs: Array<Record<string, unknown>> = [
      { matchedBy: "chargeback_alert", matchedValue: pp.data.id },
    ];
    if (record.matchedSaleId) {
      // Normal path: submission is linked to a live sale. Scope member-based match by saleId to prevent
      // false-positive against a different sale's Clawback for the same member.
      clawbackWhereOrs.push(
        { saleId: record.matchedSaleId, matchedBy: "member_id", matchedValue: record.memberId ?? "__none__" },
        { saleId: record.matchedSaleId, matchedBy: "member_name", matchedValue: record.memberCompany ?? "__none__" },
      );
    } else {
      // Phase 81-03 (UAT-005): Submission orphaned (matched_sale_id IS NULL). Original sale likely deleted +
      // re-entered. The original Clawback was cascade-deleted too, but a NEW Clawback may exist for the same
      // member via POST /clawbacks on the re-entered sale. Widen the search — unscoped by saleId.
      // Accepted risk: may false-positive to a different sale's Clawback if the same member has multiple
      // chargebacks. orderBy createdAt desc picks the most recent, which is almost always the correct one.
      if (record.memberId) {
        clawbackWhereOrs.push({ matchedBy: "member_id", matchedValue: record.memberId });
      }
      if (record.memberCompany) {
        clawbackWhereOrs.push({ matchedBy: "member_name", matchedValue: record.memberCompany });
      }
    }
    const clawback = await tx.clawback.findFirst({
      where: { OR: clawbackWhereOrs },
      include: { sale: { include: { agent: true } } },
      orderBy: { createdAt: "desc" },  // Phase 81-03 (UAT-005): prefer most recent Clawback
    });
    ```
