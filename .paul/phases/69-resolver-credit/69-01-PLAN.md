<objective>
## Goal
Extend the Phase 68 outreach leaderboards with a new `assistSaves` column so coverage reps get visible credit when they resolve another rep's assigned record as SAVED. Shift bypass rollup attribution from assignee to resolver (whoever clicked the override owns that choice). Preserve every existing Phase 68 metric identically — this is additive, not rewriting.

## Purpose
Within minutes of v2.9 launching, an agent saved another agent's assigned pending term — and got no analytics credit for it. The strict assignee-credit model from Phase 68 is the correct accountability lens for workload ownership, but incomplete for measuring performance production. Phase 69 adds the resolver-credit lens without breaking assignee-credit — both pictures become visible in one table.

## Output
- Extended aggregator with `assistSaves` per rep + resolver-keyed bypass rollup
- Upgraded leaderboard tables (CB + PT) with new column immediately after `Saved`
- Updated unit tests covering assist attribution edges (same-rep, admin resolver, roster mismatch)
- No schema changes, no new endpoints, no migrations
</objective>
