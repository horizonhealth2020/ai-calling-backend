<objective>
## Goal
Create the ContactAttempt data model and API endpoints, plus add a resolution gate that requires 3 call attempts before a chargeback or pending term can be marked as cancelled/no_contact.

## Purpose
CS agents currently click "resolve" with no accountability trail. ContactAttempt records each outreach effort (call, email, text) with timestamps, and the resolution gate enforces that agents make 3 call attempts before closing a record as lost.

## Output
- ContactAttempt Prisma model with polymorphic FK to ChargebackSubmission and PendingTerm
- POST /api/contact-attempts endpoint (create attempt, auto-calc attemptNumber)
- GET /api/contact-attempts endpoint (list by chargeback or pending term)
- Resolution gate on both resolve endpoints (3 CALL attempts required for cancelled/no_contact)
- `no_contact` added to resolution type enums
</objective>

      pendingTerm          PendingTerm?          @relation(fields: [pendingTermId], references: [id])
      agent                User                  @relation("ContactAttemptAgent", fields: [agentId], references: [id])

      @@map("contact_attempts")
    }

    **Add contact attempt endpoints in chargebacks.ts** (since both chargebacks and pending terms share the CS routes pattern):

    **POST /api/contact-attempts:**
    - Zod schema: `{ type: z.enum(["CALL", "EMAIL", "TEXT"]), notes: z.string().max(500).optional(), chargebackSubmissionId: z.string().optional(), pendingTermId: z.string().optional() }`
    - Validate exactly one of chargebackSubmissionId or pendingTermId is provided

    - Query param: chargebackSubmissionId or pendingTermId (require exactly one)

    - Include agent name in response
    - Role: requireRole("CUSTOMER_SERVICE", "SUPER_ADMIN", "OWNER_VIEW")

    **Modify resolve endpoints — add resolution gate:**

    In chargebacks.ts PATCH /chargebacks/:id/resolve:
    - Expand resolutionType enum: `z.enum(["recovered", "closed", "no_contact"])`
    - Before update: if resolutionType is "closed" or "no_contact":
      - Count ALL ContactAttempts for this chargebackSubmissionId (any type)
      - If total attempt count is 0: skip gate (pre-v2.9 record, never entered outreach workflow) <!-- audit-added -->
      - Otherwise count CALL-type ContactAttempts
      - If CALL count < 3: logAudit with action "RESOLUTION_GATE_BLOCKED" <!-- audit-added -->, return 400 `{ error: "3 call attempts required before closing. Current: N/3" }`

      - If total attempt count is 0: skip gate (pre-v2.9 record) <!-- audit-added -->

      - If CALL count < 3: logAudit with action "RESOLUTION_GATE_BLOCKED" <!-- audit-added -->, return 400 `{ error: "3 call attempts required before cancelling. Current: N/3" }`
    - "saved" bypasses gate
