<objective>
## Goal
Make the CS dashboard — chargeback/pending-term tracking (CSTracking), per-agent work queue (CSMyQueue), submission intake (CSSubmissions), resolved log (CSResolvedLog), and owner analytics (CSAnalytics) — usable on a 375px phone. Tables become responsive cards via `.responsive-table + data-label`, the inline "Work" expanded-row workspace in CSTracking (outreach logging, gate override, resolution controls) stacks cleanly on mobile with inline safe-area submit, and CSAnalytics Recharts render via existing ResponsiveContainer. No structural rewrites; attribute-only retrofit where possible with targeted form-stacking inside the expanded workspace.

## Purpose
CS agents work chargebacks from the floor and sometimes while on the phone with a customer — logging an attempt, flipping resolution to SAVED/CANCELLED/NO_CONTACT, and invoking the 3-call gate override must all be one-tap-comfortable on a phone, with no keyboard-occluded submit and no offscreen override CTA. Owners occasionally spot-check CSAnalytics on mobile; leaderboards need to read as cards, correlation chart needs to fit. Closes the v3.0 milestone.

## Output
- CSTracking: both tables (chargebacks, pending terms) responsive-table + data-label; expanded "Work" row wrapped with `.responsive-table-no-label` + internal form stacking (`.stack-mobile-md`); gate-override section elevated inline above the resolution button; submit buttons inline + safe-area padding; status filters / search row stacks on mobile
- CSMyQueue: single table responsive-table + data-label; header stacks
- CSSubmissions: both paste-review tables responsive-table + data-label; paste textarea + submit already-stacked structurally, add `.touch-target` on submit; header stacks
- CSResolvedLog: single table responsive-table + data-label; header stacks
- CSAnalytics: leaderboard + correlation + pivot tables responsive-table + data-label; Recharts wrappers unchanged (ResponsiveContainer handles width); the `minWidth: 880` leaderboard at line 903 gets horizontal-scroll wrapper + textual swipe hint (too many columns to card-ify cleanly); section headers stack on mobile
</objective>
