<objective>
## Goal
Make the manager dashboard's three highest-value surfaces — sales entry (`ManagerEntry`), today's sales list (`ManagerSales`), and the tracker + lead-timing strip (`ManagerTracker` + `LeadTimingSection`/`LeadTimingHeatmap`) — usable on a phone (375px). Forms stack and become thumb-reachable; tables become card lists; the lead-timing heatmap stays horizontally scrollable rather than collapsing into uselessness. Manager Audits (758 lines, separate tab) and Manager Config (admin) are deliberately deferred to a follow-up plan.

## Purpose
Phase 72 shipped the foundation (breakpoint hooks, MobileDrawer, touch-target/responsive utilities). Managers enter sales, watch the leaderboard, and react to lead-timing signals — those three flows must work on a phone first because the rest of the dashboards (payroll, owner, CS) target back-office desks. A manager seeing a hot lead source notification is the fastest path to sales-floor adoption of the mobile view.

## Output
- `ManagerEntry`: stacked form on mobile, full-width inputs, touch-sized controls, sticky submit bar, ACA section visually compact below 768px
- `ManagerSales`: table → card-per-sale layout on mobile via `.responsive-table` (data-label) pattern OR a dedicated mobile card render path; inline sale-edit form stays usable below 768px
- `ManagerTracker`: agent grid stacks 1-column with full-width tracker cards; key metrics readable without horizontal scroll
- `LeadTimingSection` + `LeadTimingHeatmap`: heatmap wrapped in a horizontal-scroll container with a hint affordance; `BestSourceCard` and `LeadTimingSparklines` stack vertically
- Zero regressions on desktop layout (verified via `useIsMobile().mounted` gating + `.hide-mobile`/`.show-mobile`/`.stack-mobile` utility classes — no JSX duplicated for two viewports where a class can do the work)
</objective>
